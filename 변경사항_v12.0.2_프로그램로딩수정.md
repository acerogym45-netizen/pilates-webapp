# 📌 변경사항 v12.0.2 - 프로그램 로딩 버그 수정

**날짜**: 2026-03-19  
**버전**: v12.0.2  
**작업자**: AI Assistant

---

## 🐛 발견된 버그

### **증상**
- 입주자 페이지에서 프로그램 선택 드롭다운이 비어있음
- 관리자 프로그램 관리 페이지에서도 프로그램이 표시되지 않음
- 콘솔에서 "0 programs loaded" 메시지 확인

### **원인 분석**
1. **데이터베이스 필드 불일치**:
   - `programs` 테이블의 `complex_id` 필드: `"cheongju-sk"` (complex_code 저장)
   - `complexContext.getComplexId()`: UUID 형식의 id 반환 (예: `"abc-123-def-456"`)
   - 필터 조건 불일치로 인해 데이터를 찾지 못함

2. **API 필터링 실패**:
   ```javascript
   // 기존 코드 (잘못됨)
   const programs = result.data.filter(p => p.complex_id === complexId);
   // complexId = "abc-123-def" (UUID)
   // p.complex_id = "cheongju-sk" (code)
   // → 매칭 안 됨!
   ```

---

## ✅ 해결 방법

### **1. ComplexContext에 `getComplexCode()` 메서드 추가**

**파일**: `js/complex-context.js`

```javascript
// 현재 단지 ID 가져오기
getComplexId() {
    return this.currentComplex ? this.currentComplex.id : null;
}

// 현재 단지 코드 가져오기 (프로그램 필터링 용)
getComplexCode() {
    return this.currentComplex ? this.currentComplex.complex_code : null;
}
```

**변경 이유**:
- `id`: UUID, DB 관계 및 API 호출용
- `complex_code`: "cheongju-sk" 형식, URL 파라미터 및 프로그램 필터링용

---

### **2. 입주자 페이지 프로그램 로딩 수정**

**파일**: `js/main.js` (778-805번 줄)

**변경 전**:
```javascript
const complexId = complexContext.getComplexId();
const programs = (result.data || [])
    .filter(p => p.is_active && p.complex_id === complexId)
```

**변경 후**:
```javascript
const complexCode = complexContext.getComplexCode();
const programs = (result.data || [])
    .filter(p => p.is_active && p.complex_id === complexCode)
```

**개선 사항**:
- 로그에 실제 programs 배열 출력 추가
- 에러 메시지 개선

---

### **3. 관리자 페이지 기본값 수정**

**파일**: `js/admin-programs.js` (26-38번 줄)

**변경 전**:
```javascript
async function getDefaultComplexId() {
    // ...
    return result.data[0].id; // UUID 반환
}
```

**변경 후**:
```javascript
async function getDefaultComplexId() {
    // ...
    return result.data[0].complex_code; // complex_code 반환
}
return 'cheongju-sk'; // Fallback 기본값 추가
```

## 📌 중요 참고사항

### **complex_id vs complex_code**
| 필드 | 형식 | 용도 |
|------|------|------|
| `id` | UUID | DB primary key, 관계 설정 |
| `complex_code` | 문자열 | URL 파라미터, 프로그램 필터링 |

### **프로그램 데이터 구조**
```javascript
{
  "complex_id": "cheongju-sk",     // ← complex_code 사용!
  "program_name": "화&목 6:1 그룹수업",
  "program_type": "그룹",
  "schedule_days": "화,목",
  "schedule_times": "09:00-11:00, 19:00-21:00",
  "price": 150000,
  "max_capacity": 6,
```

## 🔄 향후 개선 사항

1. **복합 키 사용 고려**:
   - `complex_id`를 UUID로 변경하고 `complex_code`를 별도 인덱스로 관리
   - 현재는 `complex_code`를 `complex_id`로 사용 (단순성 우선)

2. **에러 처리 강화**:
   - 프로그램이 없을 때 사용자 친화적 메시지 표시
   - 네트워크 오류 시 재시도 로직 추가

3. **캐싱 전략**:
   - 프로그램 데이터를 sessionStorage에 캐싱
   - 페이지 전환 시 재로딩 최소화
