# 변경사항 v12.1.3 - 문의 숨김/삭제 필터링 적용

**배포일**: 2026-03-20  
**버전**: v12.1.3  
**이전 버전**: v12.1.2  
**긴급 수정**: 입주자 페이지 문의 필터링

---

## 🚨 긴급 수정사항

관리자가 문의를 숨김 처리했는데도 입주자 페이지에 계속 표시되는 문제 수정

---

## 📋 문제 분석

### 현상
- 관리자 페이지에서 "숨김" 버튼 클릭
- 문의가 회색으로 변경되고 [숨김] 뱃지 표시
- **하지만 입주자 페이지에서 여전히 보임**

### 근본 원인
입주자 페이지 `loadPublicInquiries()` 함수에서 `is_hidden`, `is_deleted` 필터링 누락

**변경 전**:
```javascript
const publicInquiries = inquiries.filter(inq => 
    inq.is_public && 
    inq.complex_id === complexId
);
```

**문제점**:
- `is_hidden = true` → 숨김 처리된 문의도 표시됨
- `is_deleted = true` → 삭제된 문의도 표시됨

---

## 🔧 해결 방법

### 필터링 조건 추가

**변경 후**:
```javascript
const publicInquiries = inquiries.filter(inq => 
    inq.is_public &&              // 공개 문의만
    inq.complex_id === complexId && // 해당 단지만
    !inq.is_hidden &&             // 숨김 아닌 것만 ✅ NEW
    !inq.is_deleted                // 삭제 아닌 것만 ✅ NEW
);

console.log(`📋 Loaded ${publicInquiries.length} public inquiries (filtered from ${inquiries.length} total)`);
```

---

## 📊 필터링 로직

### 입주자 페이지 (index.html)
```javascript
// 4가지 조건 모두 만족해야 표시됨
✅ is_public = true       (공개 문의)
✅ complex_id 일치        (해당 단지)
✅ is_hidden = false      (숨김 아님) ← NEW
✅ is_deleted = false     (삭제 아님) ← NEW
```

### 관리자 페이지 (admin-inquiry.html)
```javascript
// 삭제된 문의만 제외
❌ is_deleted = true      (제외)
✅ 나머지 모두 표시 (숨김 포함)
```

## 🧪 테스트 시나리오

### 시나리오 1: 문의 숨김 처리

#### 1. 사전 준비
```
기존 데이터:
- 문의 1: 제목="테스트", is_public=true, is_hidden=false
```

#### 2. 관리자 작업
```
1. admin-inquiry.html → 문의 관리
2. "테스트" 문의 → "숨김" 버튼 클릭
3. 확인 → is_hidden = true로 변경
```

#### 3. 입주자 페이지 확인
```
1. index.html?complex=cheongju-sk 접속
2. 공개 문의 섹션 확인
✅ 예상: "테스트" 문의가 보이지 않음
```

#### 4. 콘솔 확인 (F12)
```javascript
// 예상 로그
📋 Loaded 0 public inquiries (filtered from 1 total)
// 1개 전체 중 필터링 후 0개
```

#### 1. 관리자 작업

```

#### 2. 입주자 페이지 확인
```
✅ 예상: "테스트" 문의가 보이지 않음

```

(loadInquiries에서 is_deleted 필터링 적용됨)
```

---

### 시나리오 3: 공개/비공개 + 숨김 조합

| is_public | is_hidden | is_deleted | 입주자 표시 | 관리자 표시 |
|-----------|-----------|------------|------------|------------|
| true | false | false | ✅ 표시 | ✅ 표시 |
| true | true | false | ❌ 숨김 | ✅ 표시 (회색) |
| true | false | true | ❌ 삭제 | ❌ 삭제 |
| false | false | false | ❌ 비공개 | ✅ 표시 |
| false | true | false | ❌ 비공개+숨김 | ✅ 표시 (회색) |

## 🐛 문제 해결

### 문제 1: 숨김 처리했는데 여전히 보임

**원인**: 브라우저 캐시

**해결**:
```
1. 강력 새로고침 (Ctrl+Shift+R / Cmd+Shift+R)
2. 시크릿 모드로 테스트
3. 콘솔 확인:
   - 📋 Loaded X public inquiries 로그 있나요?
   - X 값이 0이 아닌가요?
```

---

### 문제 2: 관리자 페이지에서도 안 보임

**원인**: 삭제(`is_deleted = true`) 처리됨

**설명**:
```
- 숨김(is_hidden): 관리자는 볼 수 있음 (회색)
- 삭제(is_deleted): 관리자도 못 봄 (완전 삭제)
```

**확인**:
```
F12 → Application → IndexedDB 또는
관리자 → 문의 관리 → 상태 필터 "전체" 선택
→ 목록에 없으면 삭제된 것
```

---

### 문제 3: 비공개 문의가 보임

**원인**: `is_public = false`인데 표시됨 (버그)

**해결**:
```
이 문제는 발생하지 않아야 합니다.
만약 발생한다면:
1. 해당 문의의 데이터 확인
2. is_public 값 확인
3. 콘솔 로그 확인
```

## 📌 참고 사항

### 숨김 vs 삭제 차이

| 기능 | is_hidden (숨김) | is_deleted (삭제) |
|------|-----------------|-------------------|
| 입주자 표시 | ❌ 안 보임 | ❌ 안 보임 |
| 관리자 표시 | ✅ 보임 (회색) | ❌ 안 보임 |
| 데이터 보존 | ✅ 유지 | ✅ 유지 (soft delete) |
| 복구 가능 | ✅ 가능 (숨김 해제) | ❌ UI에서 불가 |
| 용도 | 임시 숨김 | 영구 삭제 |

### 권장 사용법

**숨김 사용 시나리오**:
```
- 부적절한 문의 (관리 필요)
- 중복 문의
- 테스트 문의
- 임시로 보이지 않게 하고 싶을 때
```

**삭제 사용 시나리오**:
```
- 스팸 문의
- 완전히 제거해야 하는 문의
- 더 이상 필요 없는 문의
⚠️ 주의: 복구 불가!
```
