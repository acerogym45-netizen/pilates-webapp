## 🐛 문제 #1: 월간 리포트 HTTP 422 에러

### 스크린샷 분석
```
Failed to load resource: the server responded with a status of 422 ()
Contract response status: 422
❌ Error generating report: Error: Contracts fetch failed: 422
```

### 문제 원인
**HTTP 422 Unprocessable Entity**는 서버가 요청을 이해했지만 처리할 수 없다는 의미입니다.

```javascript
// ❌ 이전 코드 (admin-complex.html)
const contractsResponse = await fetch('tables/pilates_contracts?limit=10000');
const cancellationsResponse = await fetch('tables/pilates_cancellations?limit=10000');
const inquiriesResponse = await fetch('tables/pilates_inquiries?limit=10000');
```

**원인**:
- `limit=10000`이 서버의 최대 허용 한도를 초과
- 많은 REST API는 성능 및 보안상의 이유로 한 번에 가져올 수 있는 레코드 수를 제한
- 일반적으로 1000~5000개가 안전한 범위

### 해결 방법
```javascript
// ✅ 수정된 코드
const contractsResponse = await fetch('tables/pilates_contracts?limit=1000');
const cancellationsResponse = await fetch('tables/pilates_cancellations?limit=1000');
const inquiriesResponse = await fetch('tables/pilates_inquiries?limit=1000');
const programsResponse = await fetch('tables/programs?limit=1000'); // 이미 1000
```

**개선점**:
- API 요청 성공률 향상 (422 에러 → 200 OK)
- 서버 부하 감소
- 대부분의 단지는 1000개 이하의 데이터를 보유하므로 충분

### 향후 대용량 데이터 대응 (선택 사항)
만약 1000개 이상의 데이터가 필요한 경우:
```javascript
// 페이지네이션 예시
async function fetchAllData(tableName) {
    let allData = [];
    let page = 1;
    let hasMore = true;
    
    while (hasMore) {
        const response = await fetch(`tables/${tableName}?limit=1000&page=${page}`);
        const result = await response.json();
        allData = allData.concat(result.data || []);
        
        hasMore = result.data.length === 1000;
        page++;
    }
    
    return allData;
}
```

## 🔍 문제 #2: 커리큘럼 조회 실패

### 스크린샷 분석
**관리자 페이지**:
```
✅ Loaded 1 curriculums
✏️ Editing curriculum: 4월 필라테스 커리큘럼
💾 Saving curriculum: 
   {complex_id: "cheongju-sk", year: 2026, month: 4, title: "4월 필라테스 커리큘럼", ...}
✅ Curriculum saved successfully
```

**입주자 페이지**:
```
"2026년 4월 커리큘럼이 아직 등록되지 않았습니다"
```

→ **저장은 성공했는데 조회는 실패!**

### 가능한 원인들

#### 원인 1: `is_active` 필드가 false
```javascript
// 조회 조건 (js/main.js)

    c.complex_id === complexId &&
    c.year === parseInt(year) &&
    c.month === parseInt(month) &&
    c.is_active  // ⚠️ 여기서 false면 찾지 못함
);
```

**해결**: 폼 초기화 시 체크박스를 명시적으로 체크
```javascript
// ✅ 수정된 코드 (js/admin-notices.js)
function resetCurriculumForm() {
    document.getElementById('curriculumForm').reset();
    // ... 다른 초기화 ...
    
    // 활성 상태 기본값 true로 설정
    document.getElementById('curriculumActive').checked = true;
}
```

#### 원인 2: 데이터 타입 불일치
```javascript
// 조회 시
c.year === parseInt(year)  // year가 문자열 "2026"으로 저장되면 실패
```

**해결**: 저장 시 로그에 타입 출력
```javascript
// ✅ 수정된 코드 (js/admin-notices.js)
console.log('💾 Saving curriculum:', curriculumData);
console.log(`   📍 complex_id: ${curriculumData.complex_id}`);
console.log(`   📅 year: ${curriculumData.year} (type: ${typeof curriculumData.year})`);
console.log(`   📅 month: ${curriculumData.month} (type: ${typeof curriculumData.month})`);
console.log(`   ✅ is_active: ${curriculumData.is_active}`);
```

### 디버깅 로그 강화

#### ✅ 조회 시 상세 로그 추가

## 🔍 디버깅 가이드

### 월간 리포트 여전히 실패 시

#### Case 1: 다른 HTTP 에러 발생
콘솔 확인:
```
🔍 Step 1/4: Fetching contracts...
Contract response status: 404
❌ Error generating report: Error: Contracts fetch failed: 404 Not Found
```
→ **원인**: `pilates_contracts` 테이블이 존재하지 않음  
→ **해결**: 테이블 스키마 확인 및 재생성

#### Case 2: 특정 단계에서 실패
콘솔 확인:
```
🔍 Step 1/4: Fetching contracts... ✅
🔍 Step 2/4: Fetching cancellations... ✅
🔍 Step 3/4: Fetching programs... ❌
Programs response status: 500
```
→ **원인**: 서버 내부 오류  
→ **해결**: 잠시 후 다시 시도 또는 관리자에게 문의

### 커리큘럼 조회 여전히 실패 시

#### Case 1: is_active가 false로 저장됨
콘솔 확인:
```
Checking curriculum: complex=cheongju-sk, year=2026, month=4, active=false
```
→ **원인**: 체크박스가 해제된 상태로 저장  
→ **해결**: 관리자 페이지에서 해당 커리큘럼 편집 → "활성 상태" 체크 → 저장

#### Case 2: complex_id 불일치
콘솔 확인:
```
📅 Loading curriculum for 2026-4, complex: cheongju-sk
Checking curriculum: complex=other-complex, year=2026, month=4, active=true
```
→ **원인**: 다른 단지의 커리큘럼  
→ **해결**: 올바른 단지로 로그인 또는 해당 단지용 커리큘럼 등록

#### Case 3: 데이터 타입 불일치
콘솔 확인:
```
💾 Saving curriculum: {...}
   📅 year: "2026" (type: string)  ← ⚠️ 문자열로 저장됨
```
→ **원인**: 입력 필드에서 문자열로 가져옴  
→ **해결**: `parseInt()` 적용 확인 (이미 코드에 적용되어 있음)
