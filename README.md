```javascript
// ✅ 유효한 complex_code만 선택
const validComplex = result.data.find(c => 
    c.is_active &&              // 활성화된 단지
    c.complex_code &&           // complex_code 존재
    c.complex_code.length > 2 && // 최소 3글자
    c.complex_code !== 't' &&   // "t" 제외
    c.complex_code !== 'true'   // "true" 제외
);
```

**개선 사항**:
1. ✅ 활성화된 단지 우선 선택
2. ✅ `complex_code` 길이 검증 (최소 3글자)

### v12.2.6 (2026-03-20) - 커리큘럼 조회 수정 (complex_id 불일치 해결) 🐛
**핵심 버그 수정**:
- 🐛 **커리큘럼 조회 실패 원인 해결**: complex_id 불일치 문제
  - **문제**: 입주자 페이지는 UUID로 조회, 관리자는 complex_code로 저장
  - **증상**: 콘솔 로그에서 `complex_id=a3bd3a67-...` vs `complex=cheongju-sk` 불일치
  - **해결**: 입주자 페이지도 `complex_code` 사용하도록 수정

**기술적 세부사항**:
```javascript
// ❌ 이전 (입주자 페이지)
const complexId = complexContext.getComplexId();  // UUID 반환
// → 조회: complex_id='a3bd3a67-f7e0-4edb-a9bb-236347e0d5e4'

// ❌ 저장 (관리자 페이지)
complex_id: currentComplexCode  // 'cheongju-sk' 저장

// ✅ 수정 후 (입주자 페이지)
const complexId = complexContext.getComplexCode();  // 'cheongju-sk' 반환
// → 조회: complex_id='cheongju-sk' ✅ 매칭 성공!
```

```

🎯 Target curriculum found: undefined  ← 불일치로 실패

✅ 수정 후:
📅 Loading curriculum for 2026-4, complex: cheongju-sk
Checking curriculum: complex=cheongju-sk, year=2026, month=4, active=true
🎯 Target curriculum found: {...}  ← 매칭 성공!
```

**수정 파일**:
- `js/main.js` (+1 line) - `getComplexId()` → `getComplexCode()` 변경
