# v12.1.3 문의 관리 숨김/수정/삭제 기능 수정

## 🔥 긴급 버그 수정 (2024-03-20)

### 📋 변경 개요
- **문제**: 문의 관리 페이지에서 "숨김처리", "다시 표시", "수정", "삭제" 기능이 작동하지 않음
- **원인**: PUT 요청 시 시스템 필드(`gs_project_id`, `gs_table_name`, `updated_at`)를 제거하지 않아 업데이트 실패
- **해결**: 3개 함수 모두 시스템 필드 제거 로직 추가

---

## 🛠️ 수정 내역

### 1. `toggleHideInquiry()` 함수 수정
**파일**: `js/admin-inquiry.js`

```javascript
// ❌ 수정 전
const updateData = {
    ...inquiry,
    is_hidden: newStatus
};

const response = await fetch(`tables/pilates_inquiries/${inquiryId}`, {
    method: 'PUT',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(updateData)
});

// ✅ 수정 후
const updateData = {
    ...inquiry,
    is_hidden: newStatus
};

// Remove system fields that shouldn't be updated
delete updateData.gs_project_id;
delete updateData.gs_table_name;
delete updateData.updated_at;

const response = await fetch(`tables/pilates_inquiries/${inquiryId}`, {
    method: 'PUT',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(updateData)
});
```

**효과**:
- "숨김처리" 버튼 클릭 시 → `is_hidden: true` 업데이트 성공
- "다시 표시" 버튼 클릭 시 → `is_hidden: false` 업데이트 성공
- 숨겨진 문의는 회색으로 표시됨

### 3. `deleteInquiry()` 함수 수정
**파일**: `js/admin-inquiry.js`

```javascript

delete updateData.gs_project_id;
delete updateData.gs_table_name;
```

**효과**:
- "삭제" 버튼 클릭 시 소프트 삭제 성공
- 삭제된 문의는 목록에서 사라짐 (필터링됨)

## 🔍 테스트 시나리오

### ✅ 숨김/표시 토글 테스트
1. 문의 관리 페이지에서 문의 항목의 "숨김처리" 버튼 클릭
2. 확인 다이얼로그에서 "확인" 클릭
3. **예상 결과**:
   - 문의 항목이 회색으로 변경됨
   - `[숨김]` 라벨 표시됨
   - 버튼이 "다시 표시"로 변경됨
   - 콘솔 로그: `✅ Inquiry 숨김 successfully`

4. "다시 표시" 버튼 클릭
5. **예상 결과**:
   - 문의 항목이 정상 색상으로 복구됨
   - `[숨김]` 라벨 제거됨
   - 버튼이 "숨김처리"로 변경됨
   - 콘솔 로그: `✅ Inquiry 숨김 해제 successfully`

### ✅ 수정 테스트
1. "수정" 버튼 클릭
2. 제목 수정 프롬프트에서 새 제목 입력
3. 내용 수정 프롬프트에서 새 내용 입력
4. **예상 결과**:
   - 문의 목록에서 제목/내용이 즉시 업데이트됨
   - 콘솔 로그: `✅ Inquiry updated successfully`

### ✅ 삭제 테스트
1. "삭제" 버튼 클릭
2. 2번의 확인 다이얼로그에서 "확인" 클릭
3. **예상 결과**:
   - 문의 항목이 목록에서 제거됨
   - 콘솔 로그: `✅ Inquiry deleted successfully`
   - DB에는 `is_deleted: true`로 저장됨 (소프트 삭제)

> delete updateData.gs_table_name;
> delete updateData.updated_at;

## 📚 참고 사항
- 시스템 필드는 DB가 자동으로 관리하므로 클라이언트에서 업데이트하면 안 됨
- `submitReply()` 함수에는 이미 적용되어 있었으나, 신규 추가된 3개 함수에는 누락되어 있었음
