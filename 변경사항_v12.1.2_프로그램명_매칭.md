**배포일**: 2026-03-20

---

## 🚨 긴급 수정사항

v12.1.1 배포 후 **기존 신청자 정원이 여전히 [0/6명]으로 표시되는 문제** 추가 수정

---

## 📋 문제 분석

### 근본 원인
프로그램명 불일치로 인한 매칭 실패

**상황**:
```javascript
// 데이터베이스의 프로그램명 (programs 테이블)
"화&목 6:1 그룹수업(12시&18시 3.23 12:00시 오픈 예정)"

// 기존 신청자의 lesson_type (pilates_contracts 테이블)
"화&목 6:1 그룹수업"
```

**문제**:
- `programTimeSlots["화&목 6:1 그룹수업"]` → 키가 없음 (정확히는 `"화&목 6:1 그룹수업(12시&18시 3.23 12:00시 오픈 예정)"`)
- 따라서 카운트가 0으로 유지됨

---

## 🔧 해결 방법

### Fuzzy Matching 로직 추가

기존의 엄격한 매칭 (exact match) 실패 시, 유연한 매칭 (fuzzy match)을 시도합니다.

**변경 전**:
```javascript
// Exact match only
if (programTimeSlots[program] && programTimeSlots[program].hasOwnProperty(time)) {
    programTimeSlots[program][time]++;
} else {
    console.warn(`⚠️ Program "${program}" not found`);
}
```

**변경 후**:
```javascript
// Try exact match first
if (programTimeSlots[program] && programTimeSlots[program].hasOwnProperty(time)) {
    programTimeSlots[program][time]++;
    console.log(`✅ Counted: ${contract.name} - ${program} ${time}`);
} else {
    // Try fuzzy matching
    let matched = false;
    for (let programName in programTimeSlots) {
        // Check if program name includes or is included in lesson_type
        if (programName.includes(program) || program.includes(programName)) {
            if (programTimeSlots[programName].hasOwnProperty(time)) {
                programTimeSlots[programName][time]++;
                console.log(`✅ Fuzzy matched: "${program}" → "${programName}" ${time}`);
                matched = true;
                break;
            }
        }
    }
    
    if (!matched) {
        console.warn(`⚠️ No match found for "${program}"`);
    }
}
```

## 📊 매칭 예시

### 시나리오 1: Exact Match
```javascript
Program: "화&목 6:1 그룹수업"
Contract lesson_type: "화&목 6:1 그룹수업"
→ ✅ Exact match
```

### 시나리오 2: Fuzzy Match (프로그램명이 더 김)
```javascript
Program: "화&목 6:1 그룹수업(12시&18시 3.23 12:00시 오픈 예정)"
Contract lesson_type: "화&목 6:1 그룹수업"
→ ✅ Fuzzy match (programName.includes(program))
```

### 시나리오 3: Fuzzy Match (계약 데이터가 더 김)
```javascript
Program: "화&목 6:1 그룹수업"
Contract lesson_type: "화&목 6:1 그룹수업 (저녁반)"
→ ✅ Fuzzy match (program.includes(programName))
```

### 시나리오 4: No Match
```javascript
Program: "화&목 6:1 그룹수업"
Contract lesson_type: "수&금 6:1 그룹수업"
→ ⚠️ No match (완전히 다른 프로그램)
```

## 📌 권장 사항

### 프로그램명 작성 가이드

#### ✅ 좋은 예
```
- "화&목 6:1 그룹수업"
- "수&금 6:1 그룹수업"
- "월 6:1 저녁반"
- "1:1 개인레슨"
- "2:1 듀엣레슨"
```

#### ❌ 피해야 할 예
```
- "화&목 6:1 그룹수업(12시&18시 3.23 12:00시 오픈 예정)"
  → 너무 길고 복잡, 날짜/시간 정보는 별도 필드 사용
  
- "화목그룹" vs "화&목 그룹"
  → 기호(&) 사용 일관성 필요
  
- "6:1그룹수업 화목" vs "화목 6:1 그룹수업"
  → 순서 일관성 필요
```

#### 🎯 최적 형식
```
형식: [요일] [정원] [레슨타입] ([부가정보])

예시:
- 화&목 6:1 그룹수업
- 화&목 6:1 그룹수업 (저녁반)
- 월 6:1 그룹수업
- 1:1 개인레슨
```

## 🐛 알려진 제한사항

### Fuzzy Matching의 한계

```javascript
// Case 1: 의도하지 않은 매칭 가능성
Program: "화&목 6:1 그룹수업"
Contract: "화&목 6:1 그룹수업 VIP"
→ ✅ Fuzzy match (하지만 실제로는 다른 프로그램일 수 있음)

// Case 2: 순서가 다른 경우 매칭 안 됨
Program: "6:1 화&목 그룹수업"
Contract: "화&목 6:1 그룹수업"
→ ❌ No match (includes()는 순서를 고려함)
```

### 해결 방법
- **프로그램명 일관성 유지** (가장 중요!)
- 새 프로그램 추가 시 기존 명명 규칙 따르기
- 프로그램명 변경 시 기존 신청자 데이터도 함께 업데이트
