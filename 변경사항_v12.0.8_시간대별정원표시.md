# v12.0.8 - ⏰ 시간대별 정원 표시 개선

**날짜**: 2026-03-20

## 🎯 해결된 문제

### 문제 상황
사용자 보고:
> "기존에는 프로그램 종류 선택 -> 타임 선택 에서 정원이 타임 별로 보였는데 프로그램 종류에서 36/6명 이런식으로 표시되고 있어. 그리고 1:1, 2:1레슨은 정원 표시가 필요없어 별도 문의로 표시하면 돼"

#### v12.0.7의 문제점
1. **프로그램 종류 선택 단계**에서 전체 정원(`[36/6명]`)이 표시됨 → **잘못된 위치**
2. **원래 의도**: 프로그램 선택 후 → **희망 시간대 선택 단계**에서 각 시간대별 정원 표시
3. **1:1 개인레슨, 2:1 듀엣레슨**: 정원 개념이 없으므로 정원 표시 불필요

#### 예상 동작
```
1단계: 프로그램 종류 선택
   - 화/목 6인 그룹수업 (오전 09시~11시, 저녁 19시~21시) - 120,000원/월
   - 수/금 6인 그룹수업 (오전 09시~11시, 저녁 19시~21시) - 120,000원/월
   - 2:1 듀엣레슨 (오후 13시~17시, 상담 필요) - 280,000원/월 [별도 문의]
   - 1:1 개인레슨 (오후 13시~17시, 상담 필요) - 440,000원/월 [별도 문의]

2단계: 희망 시간대 선택 (그룹 수업만)
   - 오전 09시 [2/6명] 모집중
   - 오전 10시 [0/6명] 모집중
   - 오전 11시 [5/6명] ⚠️ 마감임박
   - 저녁 19시 [6/6명] 🔴 마감 (선택 불가)
   - 저녁 20시 [3/6명] 모집중
   - 저녁 21시 [1/6명] 모집중
```

## 🔍 기술적 세부사항

### 데이터 흐름

```
페이지 로드
  ↓
1. loadPrograms()
   - programs 테이블 조회 (모든 프로그램)
   - pilates_contracts 조회 (승인된 신청)
   - 프로그램별 전체 승인자 수 계산 (저장하지만 UI에 미표시)
   ↓
2. loadTimeSlotStatus()
   - programs 테이블 조회 (그룹 수업만 필터링)
   - pilates_contracts 조회 (승인된 신청, status='approved')
   - 프로그램별 시간대별 승인자 수 계산
   - window.programTimeSlots 전역 변수 저장
   ↓
3. populateProgramOptions()
   - 프로그램 종류 드롭다운 렌더링
   - 가격, 일정만 표시 (정원 제외)
   - 1:1/2:1 레슨은 [별도 문의] 표시
   ↓
4. 사용자 프로그램 선택
   ↓
5. updateTimeSlotOptions()
   - 선택된 프로그램의 시간대별 카운트 조회
   - 프로그램별 max_capacity 가져오기
   - 시간대별 정원 표시 ([현재/최대])
   - 마감된 시간대 disabled 처리
   ↓
6. 사용자 시간대 선택 및 신청 제출
   ↓
7. handlePage1Submit()
```
