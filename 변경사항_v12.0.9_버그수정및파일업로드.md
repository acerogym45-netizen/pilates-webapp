# v12.0.9 - 🐛 데이터 표시 버그 수정 및 📸 이미지 업로드 기능 추가

**날짜**: 2026-03-20

## 🎯 해결된 문제

## ✅ 해결 내용

### 🐛 문제 1: 개인 레슨 신청이 관리자 페이지에 표시 안됨

#### 근본 원인
**파일**: `js/main.js` - `submitContract()` 함수

1. **한글 상태 저장 문제** (221번 라인):
```javascript
// 변경 전
status: '승인',  // 한글로 저장
```
- v12.0.6부터 DB는 영문(`'approved'`, `'pending'`, `'rejected'`) 저장
- 관리자 페이지 필터링은 영문 기준
- 한글로 저장하면 필터링에서 제외됨

2. **complex_id UUID 사용 문제** (216번 라인):
```javascript
// 변경 전
complex_id: complexContext.getComplexId(),  // UUID 반환
```
- `getComplexId()` → UUID 반환 (예: "abc-123-def")
- DB의 `complex_id` → 문자열 코드 저장 (예: "cheongju-sk")
- 필터링 시 `complex_id` 불일치로 데이터 조회 실패

#### 해결책
```javascript
// 변경 후
const contractData = {
    ...formData,
    complex_id: complexContext.getComplexCode(),  // ← complex_code 사용
    terms_agreement: termsAgreement,
    signature: signatureName,
    signature_image: signatureImage,
    signature_date: signatureDate,
    status: 'pending',  // ← 영문 'pending' (대기중)
    created_at: Date.now()
};
```

**효과**:
- ✅ 신청 제출 시 `status: 'pending'` (영문)으로 저장
- ✅ `complex_id: "cheongju-sk"` (올바른 코드) 저장
- ✅ 관리자 페이지 "대기중" 필터에 정상 표시

### 🐛 문제 2: 강사 정보 등록 후 입주민 페이지에 표시 안됨

#### 근본 원인

**파일 1**: `instructors.html` - `loadInstructors()` 함수 (62번 라인)
```javascript
// 변경 전
const complexId = complexContext.getComplexId();  // UUID
const instructors = result.data.filter(i => 
    i.is_active && i.complex_id === complexId  // UUID와 비교
);
```
- `getComplexId()` → UUID
- DB의 `instructors.complex_id` → `complex_code` (문자열)
- 필터링 실패 → 강사 0명 조회

**파일 2**: `js/admin-instructors.js` - `saveInstructor()` 함수
```javascript
// 변경 전
const instructorData = {
    name,
    position,
    // complex_id 필드 누락!
    photo_url: photoUrl,
    ...
};
```
- `complex_id` 필드를 아예 저장하지 않음
- 필터링 시 `undefined === "cheongju-sk"` → 항상 false

#### 해결책

**파일 1**: `instructors.html`
```javascript
// 변경 후
const complexCode = complexContext.getComplexCode();  // complex_code
const instructors = result.data.filter(i => 
    i.is_active && i.complex_id === complexCode  // 올바른 비교
);
```

**파일 2**: `js/admin-instructors.js`
```javascript
// 변경 후 - 초기화 시 complex_code 로드
let currentComplexCode = null;

async function getDefaultComplexCode() {
    const response = await fetch('tables/complex_settings?...');
    const result = await response.json();
    const validComplex = result.data.find(c => c.is_active && c.complex_code);
    return validComplex.complex_code;
}

document.addEventListener('DOMContentLoaded', async function() {
    currentComplexCode = await getDefaultComplexCode();
    loadInstructors();
});

// saveInstructor 함수에서 complex_id 포함
const instructorData = {
    complex_id: currentComplexCode,  // ← 추가!
    name,
    position,
    ...
};
```

**효과**:
- ✅ 강사 등록 시 `complex_id: "cheongju-sk"` 저장
- ✅ 입주민 페이지에서 올바른 `complex_code` 기준 필터링
- ✅ 등록한 강사가 즉시 입주민 페이지에 표시

### 🆕 기능 4: 강사 프로필 이미지 파일 업로드 기능 추가

#### 기존 방식의 문제점
- URL 입력만 가능
- 외부 이미지 호스팅 서비스 필요
- 관리 불편

#### 새로운 기능

**파일 1**: `admin-instructors.html` - UI 개선
```html
<!-- 변경 전 -->
<div class="form-group">
    <label for="instructorPhotoUrl">프로필 사진 URL</label>
    <input type="text" id="instructorPhotoUrl" placeholder="https://...">
</div>

<!-- 변경 후 -->
<div class="form-group">
    <label for="instructorPhotoFile">프로필 사진 업로드</label>
    <input type="file" id="instructorPhotoFile" accept="image/*" 
           onchange="handlePhotoUpload(event)">
    <small style="color: #666;">또는</small>
</div>

<div class="form-group">
    <label for="instructorPhotoUrl">프로필 사진 URL</label>
    <input type="text" id="instructorPhotoUrl" placeholder="..." readonly>
    <small style="color: #666;">
        파일 업로드 시 자동으로 채워집니다. 비워두면 기본 아이콘이 표시됩니다
    </small>
</div>
```

**파일 2**: `js/admin-instructors.js` - 파일 업로드 처리
```javascript
// 파일을 Base64로 변환하여 URL 필드에 저장
async function handlePhotoUpload(event) {
    const file = event.target.files[0];
    if (!file) return;
    
    // 1. 파일 타입 검증 (이미지만)
    if (!file.type.startsWith('image/')) {
        alert('이미지 파일만 업로드 가능합니다.');
        return;
    }
    
    // 2. 파일 크기 검증 (최대 2MB)
    const maxSize = 2 * 1024 * 1024;
    if (file.size > maxSize) {
        alert('파일 크기는 2MB 이하여야 합니다.');
        return;
    }
    
    // 3. Base64로 변환
    const base64 = await fileToBase64(file);
    
    // 4. URL 필드에 Base64 데이터 설정
    document.getElementById('instructorPhotoUrl').value = base64;
    
    console.log('✅ Photo uploaded:', {
        name: file.name,
        size: (file.size / 1024).toFixed(2) + ' KB',
        type: file.type
    });
}

function fileToBase64(file) {
    return new Promise((resolve, reject) => {
        const reader = new FileReader();
        reader.onload = () => resolve(reader.result);
        reader.onerror = reject;
        reader.readAsDataURL(file);
    });
}
```

#### 작동 원리

1. **파일 선택** → `<input type="file">`
2. **유효성 검증** → 이미지 타입, 2MB 이하
3. **Base64 변환** → FileReader API 사용
4. **URL 필드 자동 채움** → `data:image/png;base64,iVBORw0KG...`
5. **DB 저장** → `photo_url` 필드에 Base64 문자열 저장
6. **이미지 표시** → `<img src="data:image/png;base64,...">`

#### Base64 방식의 장점
✅ **외부 호스팅 불필요**: 별도의 이미지 서버 없이 DB에 직접 저장  
✅ **간편한 관리**: 파일 업로드만으로 모든 처리 완료  
✅ **데이터 일관성**: 이미지와 데이터가 한 곳에 저장  
✅ **URL 호환성**: 기존 URL 방식과 완전 호환  

#### Base64 방식의 주의사항
⚠️ **파일 크기 제한**: 2MB 이하 (대용량 이미지는 외부 호스팅 권장)  
⚠️ **DB 용량**: Base64는 원본보다 ~33% 커짐  
⚠️ **대량 이미지**: 강사가 수십 명 이상이면 외부 CDN 고려

## 🔍 기술적 세부사항

### complex_id vs complex_code 통일

**문제**: `getComplexId()` (UUID)와 `getComplexCode()` (문자열) 혼용

**해결**: 모든 데이터 저장 및 필터링을 `complex_code` (문자열) 기준으로 통일

| 용도 | 사용 함수 | 반환 값 | 예시 |
|-----|----------|--------|------|
| 데이터 저장 | `getComplexCode()` | 문자열 | `"cheongju-sk"` |
| 데이터 필터링 | `getComplexCode()` | 문자열 | `"cheongju-sk"` |
| ~~잘못된 사용~~ | ~~`getComplexId()`~~ | ~~UUID~~ | ~~`"abc-123-def"`~~ |

### 상태(status) 값 통일

**문제**: 한글(`'승인'`, `'거부'`, `'대기중'`)과 영문(`'approved'`, `'rejected'`, `'pending'`) 혼용

**해결**: DB 저장은 영문, UI 표시는 `STATUS_MAP`으로 한글 변환

```javascript
// DB 저장 (항상 영문)
status: 'pending'    // 대기중
status: 'approved'   // 승인
status: 'rejected'   // 거부

// UI 표시 (STATUS_MAP으로 변환)
const STATUS_MAP = {
    'pending': '대기중',
    'approved': '승인',
    'rejected': '거부'
};
```

### Base64 이미지 인코딩

**과정**:
```
1. File 객체 → FileReader.readAsDataURL()
2. Base64 문자열 생성
   → "data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAA..."
3. photo_url 필드에 저장
4. <img src="{photo_url}"> 태그에서 직접 사용
```

**데이터 크기 비교**:
- 원본 JPEG (500 KB) → Base64 (~667 KB, +33%)
- 원본 PNG (1 MB) → Base64 (~1.33 MB, +33%)
- 최대 권장: 2 MB → Base64 (~2.67 MB)
