### 2. 중복 신청 방지 시스템 🚫

#### 새로운 함수 추가: `checkDuplicateApplication()`

**파일**: `js/main.js` (+48 lines)

```javascript
// 🆕 중복 신청 검사 함수
async function checkDuplicateApplication(contractData) {
    try {
        const complexCode = contractData.complex_id;
        const dong = contractData.dong;
        const ho = contractData.ho;
        const name = contractData.name;
        const lessonType = contractData.lesson_type;
        
        console.log(`🔍 Checking duplicates for: ${dong}동 ${ho}호 ${name} - ${lessonType}`);
        
        // 모든 계약 조회
        const response = await fetch('tables/pilates_contracts?limit=1000');
        const result = await response.json();
        const contracts = result.data || [];
        
        // 동일 단지 + 동일인(동/호/이름) + 동일 프로그램 + 승인 상태인 계약 찾기
        const duplicates = contracts.filter(contract => 
            contract.complex_id === complexCode &&
            contract.dong === dong &&
            contract.ho === ho &&
            contract.name === name &&
            contract.lesson_type === lessonType &&
            contract.status === '승인'  // 승인된 신청만 중복으로 간주
        );
        
        if (duplicates.length > 0) {
            console.log(`⚠️ Found ${duplicates.length} duplicate(s)`);
            return true;
        }
        
        console.log('✅ No duplicates found');
        return false;
        
    } catch (error) {
        console.error('❌ Error checking duplicates:', error);
        // 에러 발생 시 안전하게 false 반환 (중복 체크 실패 시 신청 허용)
        return false;
    }
}
```

**중복 검사 기준**:
1. ✅ **complex_id** - 동일 단지
2. ✅ **dong** - 동일 동
3. ✅ **ho** - 동일 호수
4. ✅ **name** - 동일 이름
5. ✅ **lesson_type** - 동일 프로그램
6. ✅ **status** - 승인 상태만 중복으로 간주

**안전장치**:
- 에러 발생 시 신청 허용 (false 반환)
- 대기중/거부 상태는 중복으로 간주하지 않음
- 다른 프로그램은 신청 가능

## 🔍 문제 해결 가이드

### Q1. 중복 검사가 작동하지 않아요 (중복 신청이 허용됨)
```
A: 중복 검사 조건을 확인하세요.
1. F12 → Console에서 로그 확인
2. "Checking duplicates" 로그가 있는지 확인
3. 동/호/이름/프로그램이 정확히 일치하는지 확인
4. 기존 신청의 status가 '승인'인지 확인 ('pending'은 중복 아님)
5. 대소문자, 공백 차이가 없는지 확인
```

### Q2. 정상 신청도 중복으로 차단돼요
```
A: 기존 신청을 확인하세요.
1. 관리자 페이지 → 신청 관리
2. 동일인의 기존 신청 확인
3. 기존 신청이 '승인' 상태인지 확인
4. 필요시 기존 신청 취소 후 재신청
```

### Q3. 자동 승인은 되는데 현황에 반영 안 돼요
```
A: 캐시 문제일 수 있습니다.
1. 현황 페이지에서 Ctrl+Shift+R (강력 새로고침)
2. F12 → Console에서 "Loaded X programs" 로그 확인
3. status === '승인' 필터링 확인
4. complex_id 일치 확인
```

### Q4. 관리자 페이지에서 신청 목록이 안 보여요
```
A: 필터 설정을 확인하세요.
1. 관리자 페이지 → 신청 관리
2. 상태 필터를 "전체" 또는 "승인"으로 설정
3. "대기중" 필터면 안 보임 (자동 승인되었으므로)
```

### Q5. 중복 체크 에러가 발생해요
```
A: 네트워크 문제일 수 있습니다.
1. F12 → Console에서 "Error checking duplicates" 확인
2. 네트워크 연결 확인
3. 에러 발생 시에도 신청은 허용됨 (안전장치)
4. 관리자가 사후에 중복 확인 가능
```
