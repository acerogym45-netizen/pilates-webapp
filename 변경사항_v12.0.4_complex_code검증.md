# 📌 변경사항 v12.0.4 - complex_code 유효성 검증 추가

**날짜**: 2026-03-19  
**버전**: v12.0.4  
**작업자**: AI Assistant

---

## 🐛 발견된 문제

### **콘솔 로그 분석**
```
Using complex_code: t               ← 🔴 문제!
Total programs fetched: 8
- 2:1 듀엣레슨: complex_id="cheongju-sk" ❌
- 수&금 6:1 그룹수업: complex_id="cheongju-sk" ❌
- 화&목 6:1 그룹수업: complex_id="cheongju-sk" ❌
- 1:1 개인레슨: complex_id="cheongju-sk" ❌
- 2:1 듀엣레슨: complex_id="cheongju-sk" ❌
- 수&금 6:1 그룹수업: complex_id="cheongju-sk" ❌
- 화&목 6:1 그룹수업: complex_id="cheongju-sk" ❌
Loaded 0 programs for complex t    ← 🔴 매칭 실패!
```

### **원인 분석**
1. `complex_settings` 테이블의 첫 번째 레코드의 `complex_code` 필드가 **`"t"`** (한 글자)로 잘못 저장됨
2. 프로그램들의 `complex_id`는 모두 **`"cheongju-sk"`**
3. `"t" !== "cheongju-sk"` → **모든 프로그램 필터링 실패**
4. 결과: "등록된 프로그램이 없습니다" 표시

### **왜 이런 일이 발생했나?**
- `complex_settings` 테이블에 데이터를 추가할 때 `complex_code` 필드에 실수로 `"t"` 또는 `true` 같은 잘못된 값이 입력됨
- 또는 테이블 스키마 업데이트 시 기본값이 잘못 설정됨

---

## ✅ 해결 방법

### **1. complex_code 유효성 검증 로직 추가**

**파일**: `js/admin-programs.js`

**변경 전**:
```javascript
async function getDefaultComplexId() {
    const response = await fetch(`${TABLES_ENDPOINT}/complex_settings?limit=1`);
    const result = await response.json();
    
    if (result.data && result.data.length > 0) {
        const complexCode = result.data[0].complex_code;
        return complexCode; // "t" 같은 잘못된 값도 그대로 사용
    }
    return 'cheongju-sk';
}
```

**변경 후**:
```javascript
async function getDefaultComplexId() {
    try {
        console.log('🔍 Fetching default complex...');
        const response = await fetch(`${TABLES_ENDPOINT}/complex_settings?limit=100&sort=display_order`);
        const result = await response.json();
        console.log('📦 Complex settings result:', result);
        
        if (result.data && result.data.length > 0) {
            // ✅ Find first active complex with VALID complex_code
            const validComplex = result.data.find(c => 
                c.is_active &&                     // 활성화된 단지
                c.complex_code &&                  // complex_code 존재
                c.complex_code.length > 2 &&       // 최소 3글자 이상
                c.complex_code !== 't' &&          // "t" 제외
                c.complex_code !== 'true'          // "true" 제외
            );
            
            if (validComplex) {
                const complexCode = validComplex.complex_code;
                console.log(`✅ Using complex_code: ${complexCode}`);
                return complexCode;
            }
            
            // Fallback to first complex_code if valid
            const firstCode = result.data[0].complex_code;
            if (firstCode && firstCode.length > 2) {
                console.log(`⚠️ Using first complex_code: ${firstCode}`);
                return firstCode;
            }
        }
    } catch (error) {
        console.error('❌ Error getting default complex:', error);
    }
    console.log('⚠️ Using fallback: cheongju-sk');
    return 'cheongju-sk';
}
```

### **개선 사항**
1. ✅ **활성화된 단지 우선**: `is_active === true`인 단지부터 검색
2. ✅ **유효성 검증**: `complex_code.length > 2` (최소 3글자)
3. ✅ **잘못된 값 필터링**: `"t"`, `"true"` 같은 값 제외
4. ✅ **모든 단지 검색**: `limit=100`으로 전체 단지 조회
5. ✅ **정렬 적용**: `sort=display_order`로 우선순위 순서
6. ✅ **폴백 보장**: 모든 검증 실패 시 `"cheongju-sk"` 사용

1. `complex_code`가 `"t"` 또는 `"true"`인 레코드 찾기

3. 올바른 값으로 업데이트
