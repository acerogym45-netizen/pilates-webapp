# 📋 변경사항 v12.0.1 – 기존 프로그램 데이터 마이그레이션

**날짜**: 2026-03-19  
**버전**: v12.0.1  
**작업 유형**: 데이터 마이그레이션, 하드코딩 제거

---

## ✨ 주요 변경사항

### 🔄 문제 상황

**발견된 이슈**:
- 입주자 페이지: 하드코딩된 4개 프로그램 표시 (화&목, 수&금, 듀엣, 개인)
- 프로그램 관리 페이지: 데이터베이스가 비어있어 "등록된 프로그램이 없습니다" 표시
- 두 시스템이 분리되어 관리 불가능

**원인**:
- v12.0.0에서 프로그램 관리 시스템을 구축했지만 기존 데이터를 마이그레이션하지 않음
- `index.html`에 하드코딩된 프로그램 목록이 그대로 남아있음

---

## 🔄 동작 플로우

### Before (v12.0.0) - 문제 상황
```
입주자 페이지:
    ↓
하드코딩된 4개 프로그램 표시 ✅
    ↓
데이터베이스와 무관

프로그램 관리 페이지:
    ↓
데이터베이스에서 로드 시도
    ↓
"등록된 프로그램이 없습니다" ❌

→ 두 시스템이 분리됨
→ 관리자가 프로그램 수정해도 입주자 페이지 변경 안 됨
```

### After (v12.0.1) - 해결
```
데이터베이스:
    ↓
4개 프로그램 등록 ✅

입주자 페이지:
    ↓
데이터베이스에서 프로그램 로드 ✅
    ↓
4개 프로그램 표시

프로그램 관리 페이지:
    ↓
데이터베이스에서 프로그램 로드 ✅
    ↓
4개 프로그램 카드 표시

→ 하나의 데이터 소스 (Single Source of Truth)
→ 관리자가 수정하면 입주자 페이지 즉시 반영 ✅
```

## 💡 향후 관리 방법

### 프로그램 수정
```
1. admin-programs.html 접속
2. 수정할 프로그램 카드에서 "수정" 버튼 클릭
3. 정보 수정 (요금, 시간, 정원 등)
4. 저장
5. 입주자 페이지에서 즉시 반영됨
```

### 프로그램 추가
```
1. admin-programs.html 접속
2. "프로그램 추가" 버튼 클릭
3. 프로그램 정보 입력
4. 저장
5. 입주자 페이지에서 즉시 표시됨
```

### 프로그램 비활성화 (시즌 오프 등)
```
1. admin-programs.html 접속
2. 해당 프로그램 카드에서 "수정" 버튼 클릭
3. "활성화 여부"를 "비활성화"로 변경
4. 저장
5. 입주자 페이지에서 숨김 처리됨
```
