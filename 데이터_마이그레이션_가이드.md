## 📌 개요

### 현재 시스템 테이블 구조

| 테이블 | 필드 수 | 주요 필드 | 용도 |
|--------|---------|----------|------|
| `pilates_contracts` | 15 | id, dong, ho, name, phone, lesson_type, status | 레슨 신청/계약 |
| `pilates_cancellations` | 12 | id, dong, ho, name, phone, lesson_type, reason | 해지 신청 |
| `pilates_inquiries` | 13 | id, dong, ho, name, phone, title, content, reply | 문의사항 |
| `instructors` | 11 | id, name, position, photo_url, certifications | 강사 정보 |
| `notices` | 8 | id, title, content, category, is_active | 공지사항 |
| `programs` | 13 | id, program_name, price, schedule_days | 프로그램 정보 |
| `complex_settings` | 13 | id, complex_name, complex_code, is_active | 단지 설정 |
| `guestbook` | 5 | id, author, message, replies | 방명록 (선택) |

### 🆕 새로운 필드: `complex_id`

**중요**: v12.0.0부터 **다중 단지 지원**을 위해 대부분의 테이블에 `complex_id` 필드가 추가되었습니다.

기존 데이터 이관 시 반드시 `complex_id` 값을 지정해야 합니다:
- **기본값**: `"cheongju-sk"` (청주SK뷰자이)
- **형식**: `complex_code` (예: `"gangnam-hillstate"`, `"seoul-raemian"`)

### 방법 비교

| 방법 | 난이도 | 속도 | 데이터 양 | 권장 대상 |
|------|--------|------|-----------|----------|

- **데이터 < 50건**: 방법 1 (Excel/CSV)
- **데이터 50~500건**: 방법 3 (API)
- **데이터 > 500건**: 방법 2 (DB 복사)

## 방법 1: Excel/CSV 수동 업로드

### ✅ 장점
- 프로그래밍 지식 불필요
- 데이터 확인/편집 쉬움
- 안전 (실수 복구 가능)

### ⚠️ 단점
- 대량 데이터 처리 느림
- 수작업 필요

---

### 📝 Step 1: 기존 데이터 내보내기

#### **A. 기존 시스템이 Excel/CSV 제공하는 경우**
1. 기존 시스템 관리자 페이지 접속
2. 각 항목(신청, 해지, 문의 등) 페이지에서 **"CSV 다운로드"** 버튼 클릭
3. 다운로드된 CSV 파일 저장

#### **B. 기존 시스템이 DB만 있는 경우**
```sql
-- 예시: MySQL/PostgreSQL
SELECT * FROM pilates_contracts INTO OUTFILE '/tmp/contracts.csv'
FIELDS TERMINATED BY ',' ENCLOSED BY '"' LINES TERMINATED BY '\n';
```

또는:
```bash
# MySQL
mysql -u username -p database_name -e "SELECT * FROM pilates_contracts" > contracts.csv

# PostgreSQL
psql -U username -d database_name -c "\COPY pilates_contracts TO 'contracts.csv' CSV HEADER"
```

### 📊 Step 2: 데이터 형식 변환 (Excel)

#### **신청 데이터 (pilates_contracts) 예시**

**기존 CSV 형식**:
```csv
id,dong,ho,name,phone,lesson_type,preferred_time,status,created_at
1,101,201,홍길동,010-1234-5678,화&목 6:1,09:00-11:00,승인,2024-03-01
2,102,301,김철수,010-2345-6789,수&금 6:1,19:00-21:00,대기,2024-03-02
```

**새 시스템 형식 (complex_id 추가)**:
```csv
complex_id,dong,ho,name,phone,lesson_type,preferred_time,status,created_at
cheongju-sk,101,201,홍길동,010-1234-5678,화&목 6:1 그룹수업,09:00-11:00,approved,2024-03-01
cheongju-sk,102,301,김철수,010-2345-6789,수&금 6:1 그룹수업,19:00-21:00,pending,2024-03-02
```

**변경 사항**:
1. ✅ `complex_id` 열 추가 → 모든 행에 `"cheongju-sk"` 입력
2. ✅ `status` 값 변환:
   - `승인` → `approved`
   - `대기` → `pending`
   - `거절` → `rejected`
3. ✅ `lesson_type` 이름 확인 (프로그램 테이블과 일치해야 함)

### 🔄 Step 3: 브라우저 콘솔에서 데이터 업로드

새 시스템에는 CSV 업로드 UI가 없으므로 **브라우저 콘솔**을 사용합니다.

#### **업로드 스크립트**

```javascript
// 1. 새 시스템 관리자 페이지 접속 (예: admin-main.html)

// ===== 신청 데이터 업로드 =====

    name: "홍길동",

    preferred_time: "09:00-11:00",
    agreement: true,
    terms_agreement: true,
    signature: "홍길동",
    signature_date: "2024-03-01",
    status: "approved",
    created_at: new Date("2024-03-01").getTime()
  },
  {
    complex_id: "cheongju-sk",
    dong: "102",
    ho: "301",
    name: "김철수",
    phone: "010-2345-6789",
    lesson_type: "수&금 6:1 그룹수업",
    preferred_time: "19:00-21:00",
    agreement: true,
    terms_agreement: true,
    signature: "김철수",
    signature_date: "2024-03-02",
    status: "pending",
    created_at: new Date("2024-03-02").getTime()
  }
  // ... 더 많은 데이터 추가
];

// 업로드 함수

  for (let i = 0; i < contractsData.length; i++) {
    try {
      const response = await fetch('tables/pilates_contracts', {
        method: 'POST',

        body: JSON.stringify(contractsData[i])
      });
      const result = await response.json();
      console.log(`✅ ${i+1}/${contractsData.length}: ${contractsData[i].name} 업로드 완료`);
    } catch (error) {
      console.error(`❌ ${i+1}/${contractsData.length}: ${contractsData[i].name} 실패`, error);
    }
    // 1초 대기 (서버 부하 방지)
    await new Promise(resolve => setTimeout(resolve, 1000));
  }
  console.log('🎉 모든 데이터 업로드 완료!');
}

// 실행
uploadContracts();
```

### 절차

1. **기존 DB에서 데이터 추출**
```sql
SELECT * FROM pilates_contracts;
```

```sql
-- 모든 데이터에 complex_id 추가
UPDATE pilates_contracts SET complex_id = 'cheongju-sk' WHERE complex_id IS NULL;
UPDATE pilates_cancellations SET complex_id = 'cheongju-sk' WHERE complex_id IS NULL;
-- ... (모든 테이블 반복)
```

3. **새 시스템 DB에 삽입**

## 방법 3: API를 통한 프로그래밍 방식

### Python 스크립트 예시

```python
import requests
import csv
import time

API_BASE = 'https://your-domain.com/tables'
COMPLEX_ID = 'cheongju-sk'

# CSV 파일 읽기
with open('contracts.csv', 'r', encoding='utf-8') as f:
    reader = csv.DictReader(f)
    contracts = list(reader)

# 업로드
for i, contract in enumerate(contracts):
    # complex_id 추가
    contract['complex_id'] = COMPLEX_ID
    
    # status 변환
    status_map = {'승인': 'approved', '대기': 'pending', '거절': 'rejected'}
    contract['status'] = status_map.get(contract['status'], 'pending')
    
    # API 호출
    try:
        response = requests.post(
            f'{API_BASE}/pilates_contracts',
            json=contract,
            headers={'Content-Type': 'application/json'}
        )
        print(f"✅ {i+1}/{len(contracts)}: {contract['name']} 업로드 완료")
    except Exception as e:
        print(f"❌ {i+1}/{len(contracts)}: 실패 - {e}")
    
    # 1초 대기
    time.sleep(1)

print("🎉 모든 데이터 업로드 완료!")
```

## ❗ 문제 해결

### 문제 1: "complex_id가 없습니다" 오류
```
❌ Error: complex_id is required
```

**해결**: 모든 데이터에 `complex_id: "cheongju-sk"` 추가

---

### 문제 2: 날짜 형식 오류
```
❌ Error: Invalid date format
```

**해결**: `created_at`을 밀리초 타임스탬프로 변환
```javascript
created_at: new Date("2024-03-01").getTime()  // 1709251200000
```

---

### 문제 3: 중복 데이터
```
❌ Error: Duplicate entry
```

**해결**: 
1. 기존 데이터 확인 후 삭제
2. 또는 `id` 필드 제거 (자동 생성됨)

---

### 문제 4: 프로그램명 불일치
```
❌ Warning: lesson_type not found in programs
```

**해결**: 
1. 프로그램 관리 페이지에서 프로그램 확인
2. `lesson_type`을 정확한 프로그램명으로 수정
