# 📌 변경사항 v12.0.6 - 상태 필터링 로직 통일

**날짜**: 2026-03-20  
**버전**: v12.0.6  
**작업자**: AI Assistant

---

## 🐛 발견된 문제

### **증상**
- CSV로 업로드한 데이터가 관리자 페이지에서 필터링 안 됨
- 통계 카드에 `승인 2개, 거부 0개` 표시되지만 실제로는 `승인 2개, 거부 2개`
- 상태 필터를 `승인` 또는 `거부`로 선택해도 데이터가 표시되지 않음

### **원인 분석**

**데이터 불일치**:
```
┌─────────────────────┬──────────────────────┬──────────┐
│ 저장 위치           │ 저장 형식            │ 값       │
├─────────────────────┼──────────────────────┼──────────┤
│ 데이터베이스 (DB)   │ 영문                │ approved │
│                     │                     │ rejected │
│                     │                     │ pending  │
├─────────────────────┼──────────────────────┼──────────┤
│ HTML 필터           │ 한글                │ 승인     │
│                     │                     │ 거부     │
│                     │                     │ 대기중   │
├─────────────────────┼──────────────────────┼──────────┤
│ JavaScript 비교     │ 한글 (기존)         │ 승인     │
│                     │                     │ 거부     │
│                     │                     │ 대기중   │
└─────────────────────┴──────────────────────┴──────────┘

❌ 결과: "승인" !== "approved" → 필터링 실패!
```

**문제 코드**:
```javascript
// ❌ 기존 코드
const statusMatch = statusFilter === 'all' || contract.status === statusFilter;
// statusFilter = "승인" (한글)
// contract.status = "approved" (영문)
// → false! 매칭 실패
```

---

## ✅ 해결 방법

### **1. STATUS_MAP 매핑 객체 추가**

```javascript
// Status mapping: 한글 ↔ 영문
const STATUS_MAP = {
    '대기중': 'pending',
    '승인': 'approved',
    '거부': 'rejected',
    'pending': '대기중',
    'approved': '승인',
    'rejected': '거부'
};
```

**역할**:
- 한글 → 영문: 필터링 시 사용
- 영문 → 한글: 화면 표시 시 사용

---

### **2. 필터링 로직 수정**

**변경 전**:
```javascript
function filterContracts() {
    const statusFilter = document.getElementById('statusFilter').value;
    // ...
    
    filteredContracts = allContracts.filter(contract => {
        const statusMatch = statusFilter === 'all' || contract.status === statusFilter;
        // ❌ "승인" !== "approved" → 매칭 실패
        // ...
    });
}
```

**변경 후**:
```javascript
function filterContracts() {
    const statusFilter = document.getElementById('statusFilter').value;
    // ...
    
    filteredContracts = allContracts.filter(contract => {
        // ✅ 한글 필터를 영문으로 변환하여 비교
        const statusFilterEn = statusFilter === 'all' ? 'all' : (STATUS_MAP[statusFilter] || statusFilter);
        const statusMatch = statusFilterEn === 'all' || contract.status === statusFilterEn;
        // ✅ "approved" === "approved" → 매칭 성공!
        // ...
    });
}
```

---

### **3. 화면 표시 로직 수정**

**변경 전**:
```javascript
function createContractHTML(contract) {
    const statusClass = contract.status === '대기중' ? 'pending' : 
                       contract.status === '승인' ? 'approved' : 'rejected';
    // ❌ contract.status = "approved"이므로 statusClass = 'rejected' (잘못된 결과)
    
    return `
        <span class="status-badge ${statusClass}">${contract.status}</span>
        // ❌ "approved" 그대로 표시 (영문)
    `;
}
```

**변경 후**:
```javascript
function createContractHTML(contract) {
    // ✅ DB의 영문 상태를 한글로 변환
    const statusKo = STATUS_MAP[contract.status] || contract.status;
    const statusClass = contract.status === 'pending' ? 'pending' : 
                       contract.status === 'approved' ? 'approved' : 'rejected';
    // ✅ contract.status = "approved"이므로 statusClass = 'approved' (정확함)
    
    return `
        <span class="status-badge ${statusClass}">${statusKo}</span>
        // ✅ "승인" 표시 (한글)
    `;
}
```

### **4. 통계 업데이트 로직 수정**

**변경 전**:
```javascript
function updateStats() {
    const pending = allContracts.filter(c => c.status === '대기중').length;
    const approved = allContracts.filter(c => c.status === '승인').length;
    const rejected = allContracts.filter(c => c.status === '거부').length;
    // ❌ DB에는 'approved', 'rejected'로 저장되어 있으므로 0개씩 카운트됨
}
```

**변경 후**:
```javascript
function updateStats() {
    const pending = allContracts.filter(c => c.status === 'pending').length;
    const approved = allContracts.filter(c => c.status === 'approved').length;
    const rejected = allContracts.filter(c => c.status === 'rejected').length;
    // ✅ 영문으로 비교하여 정확한 개수 카운트
}
```

## 🔄 데이터 흐름

### **신규 신청 (입주자 페이지)**
```
1. 입주자가 신청서 작성
2. 서버에 저장: status = "pending" (영문)
3. 관리자 페이지 로딩
4. JavaScript에서 "대기중" 으로 표시 (한글)
5. 관리자가 "승인" 버튼 클릭
6. 서버에 업데이트: status = "approved" (영문)
7. 화면에 "승인" 표시 (한글)
```

### **CSV 업로드**
```
1. CSV 파일 업로드 (상태: "승인", "거부")
2. csv-import-tool.html에서 자동 변환
   - "승인" → "approved"
   - "거부" → "rejected"
3. 서버에 저장: "approved", "rejected" (영문)
4. 관리자 페이지 로딩
5. JavaScript에서 "승인", "거부"로 표시 (한글)
```

## 💡 설계 결정

### **왜 DB는 영문으로 저장하나요?**

**장점**:
1. ✅ **국제 표준**: 다국어 지원 가능
2. ✅ **API 호환성**: RESTful API 표준 (approved, pending, rejected)
3. ✅ **오타 방지**: "승인", "수인", "송인" 같은 오타 방지
4. ✅ **정렬/검색 안정성**: UTF-8 인코딩 문제 없음

**단점**:
1. ⚠️ 한글 UI와 불일치 → **STATUS_MAP으로 해결**
