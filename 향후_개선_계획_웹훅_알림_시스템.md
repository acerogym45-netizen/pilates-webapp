# 🔮 향후 개선 계획 - 웹훅 알림 시스템

## 📋 개요

**목표**: 관리자가 문의에 답변을 등록하면 입주자에게 자동으로 알림을 전송하여 업무 효율성을 향상시킵니다.

---

## 📌 현재 상황 (v12.1.4)

### 문제점:
1. 관리자가 답변을 등록해도 입주자는 알 수 없음
2. 관리자가 직접 전화하거나 문자를 보내야 함
3. 입주자가 직접 페이지에 방문해야만 답변 확인 가능

### 개선 필요:
- ✅ 답변 등록 시 자동 알림 전송
- ✅ 관리자 업무량 감소
- ✅ 입주자 만족도 향상

## 🎯 구현 계획: v12.2.0 - 웹훅 알림 시스템

### Phase 1: 이메일 자동 발송 (무료, 즉시 구현 가능)

#### 사용 기술:
- **Make.com** (구 Integromat) - 무료 플랜 월 1,000회
- **Gmail** - 무료

#### 구현 단계:

**1. 클라이언트 코드 수정** (`js/admin-inquiry.js`)

```javascript
// 답변 등록 성공 후 웹훅 전송
async function submitReply(event) {
    // ... 기존 코드 ...
    
    if (response.ok) {
        const result = await response.json();
        console.log('✅ Reply submitted successfully:', result);
        
        // 🔔 웹훅 전송
        await sendWebhook({
            event: 'inquiry_reply',
            inquiry_id: inquiryId,
            inquiry_title: inquiry.title,
            user_name: inquiry.name,
            user_phone: inquiry.phone,
            user_email: inquiry.email || '',
            reply_content: replyContent,
            reply_date: new Date().toISOString(),
            page_url: `https://your-domain.com/?complex=${inquiry.complex_id}#inquiries`
        });
        
        alert('답변이 등록되었습니다.');
    }
}

// 웹훅 전송 함수
async function sendWebhook(data) {
    const webhookUrl = 'https://hook.make.com/YOUR_WEBHOOK_ID';
    
    try {
        await fetch(webhookUrl, {
            method: 'POST',
            headers: { 'Content-Type': 'application/json' },
            body: JSON.stringify(data)
        });
        console.log('🔔 Webhook sent successfully');
    } catch (error) {
        console.error('❌ Webhook failed:', error);
        // 웹훅 실패해도 답변 등록은 성공
    }
}
```

**2. Make.com 설정**

1. https://www.make.com/ 가입 (무료)
2. 새 시나리오 생성
3. "Webhooks" 모듈 추가 → "Custom webhook"
4. 웹훅 URL 복사 → 위 코드에 붙여넣기
5. "Gmail" 모듈 추가 → "Send an Email"
6. 설정:
   ```
   To: {{user_email}}
   Subject: [SK뷰자이 필라테스] "{{inquiry_title}}" 문의 답변
   Body:
   안녕하세요 {{user_name}}님,
   
   문의하신 내용에 답변이 등록되었습니다.
   
   답변 내용:
   {{reply_content}}
   
   자세한 내용: {{page_url}}
   ```

**3. 비용**: **완전 무료** ✅

## 🔐 보안 고려사항

### 웹훅 URL 보호:
```javascript
// 환경변수 또는 설정 파일에서 관리
const WEBHOOK_CONFIG = {
    enabled: true,
    url: 'https://hook.make.com/YOUR_WEBHOOK_ID',
    secret: 'YOUR_SECRET_KEY'  // 선택사항
};

// HMAC 서명 추가 (선택)
async function generateSignature(data, secret) {
    const message = JSON.stringify(data);
    const encoder = new TextEncoder();
    const key = await crypto.subtle.importKey(
        'raw',
        encoder.encode(secret),
        { name: 'HMAC', hash: 'SHA-256' },
        false,
        ['sign']
    );
    const signature = await crypto.subtle.sign(
        'HMAC',
        key,
        encoder.encode(message)
    );
    return btoa(String.fromCharCode(...new Uint8Array(signature)));
}
```

## 🆘 자주 묻는 질문 (FAQ)

### Q1. 웹훅이 실패하면 답변 등록도 실패하나요?
```
A: 아니오, 웹훅 실패는 답변 등록에 영향을 주지 않습니다.
- 답변은 DB에 정상 저장됨
- 웹훅 실패 시 에러만 콘솔에 로그
- try-catch로 안전하게 처리
```

### Q2. Make.com 무료 플랜이 충분한가요?
```
A: 네, 대부분의 경우 충분합니다.
- 무료 플랜: 월 1,000회 작업
- 웹훅 1회 + 이메일 1회 = 2회 작업
- 월 500건 답변까지 무료 사용 가능
- 평균적으로 월 50-100건 답변 예상
```

### Q3. 이메일 주소가 없는 입주자는 어떻게 하나요?
```
A: SMS를 추가로 구현하거나, 전화번호만으로 알림 가능
- 이메일 없으면 SMS로 자동 전환 (Make.com 조건부 라우팅)
- 또는 문의 작성 시 이메일 필수 입력으로 변경
```

### Q4. 스팸으로 분류될 가능성은 없나요?
```
A: Gmail 발송 시 스팸 가능성 낮음
- Make.com이 공식 Gmail API 사용
- 발신자 주소를 관리자 Gmail로 설정
- 입주자가 문의 작성 시 동의 받은 상태
- 필요시 "문의 작성 시 알림 수신 동의" 체크박스 추가
```

### Q5. 배포가 복잡하지 않나요?
```
A: 매우 간단합니다!
1. Make.com 가입 (5분)
2. 시나리오 설정 (10분)
3. 코드 수정 (5분)
4. 배포 (1분)
5. 테스트 (5분)

총 소요 시간: 약 30분
```
