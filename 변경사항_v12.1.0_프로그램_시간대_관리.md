# 변경사항 v12.1.0 - 프로그램별 시간대 관리 및 문의사항 강화

**배포일**: 2026-03-20  
**버전**: v12.1.0  
**이전 버전**: v12.0.11

---

## 📋 변경 개요

사용자 요청사항을 반영하여 **프로그램별 독립 시간대 설정**, **비활성 프로그램 노출 옵션**, **문의사항 관리 기능 강화**를 구현했습니다.

### 주요 요구사항
1. ✅ 프로그램별 개설 시간대 개별 설정 (예: 화&목은 09,10,11,19시만, 수&금은 09,12,18시만)
2. ✅ 비활성화 프로그램도 입주자 페이지 노출 가능 (예고/공지용)
3. ✅ 문의사항 삭제/숨김/수정 기능
4. ✅ 신규 프로그램 추가 지원 (월 6:1 저녁반, 12시/18시 추가 시간대)

## 🔧 핵심 변경사항

### 1. 데이터베이스 스키마 확장

#### **programs 테이블** (13 → 15 fields)
```javascript
{
  // ...기존 필드
  display_on_inactive: bool,      // NEW: 비활성 상태에서도 입주자 페이지 노출 여부
  available_time_slots: array,    // NEW: ["09:00", "10:00", "11:00", "12:00", "18:00", "19:00"]
}
```

**사용 사례**:
- `display_on_inactive = true` + `is_active = false` → "3월부터 저녁반 오픈 예정!" 같은 예고 표시
- `available_time_slots = ["09:00", "10:00", "19:00"]` → 입주자는 이 3개 시간만 선택 가능

#### **pilates_inquiries 테이블** (13 → 15 fields)
```javascript
{
  // ...기존 필드
  is_hidden: bool,     // NEW: 숨김 처리 (입주자에게 보이지 않음, 관리자는 확인 가능)
  is_deleted: bool     // NEW: 삭제 처리 (soft delete, 복구 불가)
}
```

```javascript
// 3. 시간 코드를 표시명으로 변환
const timeDisplayMap = {

    '10:00': '오전 10시',
    '11:00': '오전 11시',

    '13:00': '오후 13시',
    '14:00': '오후 14시',
    '15:00': '오후 15시',
    '16:00': '오후 16시',
    '17:00': '오후 17시',
    '18:00': '저녁 18시',
    '19:00': '저녁 19시',
    '20:00': '저녁 20시',
    '21:00': '저녁 21시'
};

// 4. 설정된 시간대만 드롭다운에 표시
availableTimeSlots.forEach(timeCode => {
    const timeDisplay = timeDisplayMap[timeCode] || `${timeCode}시`;
    const count = slots[timeDisplay] || 0;
    const isFull = count >= maxCapacity;
    
    optionsHTML += `<option value="${timeDisplay}" ${disabled} ${style}>
        ${timeDisplay} [${count}/${maxCapacity}명] ${status}
    </option>`;
});
```

##### 📌 toggleHideInquiry 함수
```javascript
async function toggleHideInquiry(inquiryId, currentHiddenStatus) {
    const newStatus = !currentHiddenStatus;
    
    // 확인 모달
    if (!confirm(`이 문의를 ${newStatus ? '숨김' : '숨김 해제'} 처리하시겠습니까?`)) {
        return;
    }
    
    // API 요청
    const updateData = { ...inquiry, is_hidden: newStatus };
    const response = await fetch(`tables/pilates_inquiries/${inquiryId}`, {
        method: 'PUT',
        body: JSON.stringify(updateData)
    });
    
    if (response.ok) {
        alert(`문의가 ${action} 처리되었습니다.`);
        await loadInquiries(); // 목록 새로고침
    }
}
```

**동작**:
- `is_hidden = true` → 입주자 페이지 문의 목록에서 숨김
- 관리자 페이지에는 회색으로 표시 (`.hidden-inquiry` 클래스)

##### 📌 editInquiryContent 함수
```javascript
async function editInquiryContent(inquiryId) {
    const newTitle = prompt('새 제목을 입력하세요:', inquiry.title);
    if (newTitle === null) return;
    
    const newContent = prompt('새 내용을 입력하세요:', inquiry.content);
    if (newContent === null) return;
    
    const updateData = {
        ...inquiry,
        title: newTitle.trim(),
        content: newContent.trim()
    };
    
    // PUT 요청으로 수정
    await fetch(`tables/pilates_inquiries/${inquiryId}`, {
        method: 'PUT',
        body: JSON.stringify(updateData)
    });
}
```

##### 📌 deleteInquiry 함수 (Soft Delete)
```javascript
async function deleteInquiry(inquiryId) {
    // 이중 확인
    if (!confirm('⚠️ 이 문의를 삭제하시겠습니까?\n삭제된 문의는 복구할 수 없습니다.')) {
        return;
    }
    
    if (!confirm('정말로 삭제하시겠습니까? (복구 불가)')) {
        return;
    }
    
    const updateData = { ...inquiry, is_deleted: true };
    
    await fetch(`tables/pilates_inquiries/${inquiryId}`, {
        method: 'PUT',
        body: JSON.stringify(updateData)
    });
}
```

##### 📌 loadInquiries 필터링 수정
```javascript
async function loadInquiries() {
    const response = await fetch(`tables/pilates_inquiries?sort=-created_at&limit=1000`);
    const result = await response.json();
    
    // 삭제된 문의 제외
    allInquiries = (result.data || []).filter(inquiry => !inquiry.is_deleted);
    
    console.log(`📊 Loaded ${allInquiries.length} inquiries (excluding deleted)`);
}
```

## 🧪 테스트 시나리오

### 시나리오 1: 프로그램별 시간대 설정
```
1. 관리자 페이지 → 프로그램 관리 → "화&목 6:1 그룹수업" 수정
2. 시간대 체크박스에서 09:00, 10:00, 12:00만 선택
3. 저장 → 입주자 페이지 확인
4. 프로그램 선택 시 "오전 09시, 오전 10시, 오후 12시"만 드롭다운에 표시
```

### 시나리오 2: 비활성 프로그램 예고
```
1. 관리자 페이지 → 프로그램 추가
   - 프로그램명: "3월 신규 저녁반 (곧 오픈)"
```

## 🔍 문제 해결 (Troubleshooting)

### 문제 1: 시간대가 표시되지 않음
**증상**: 프로그램 선택 후 드롭다운이 비어있음  
**원인**: available_time_slots 데이터가 없음  
**해결**:
```
1. F12 → 콘솔 확인
2. "No time slots configured" 경고 확인
3. 관리자 페이지 → 해당 프로그램 수정
4. 시간대 최소 1개 선택 후 저장
```

### 문제 2: 문의 삭제 후에도 보임
**증상**: 삭제한 문의가 계속 표시됨  
**원인**: 페이지 캐시 또는 필터링 미적용  
**해결**:
```
1. 강력 새로고침 (Ctrl+Shift+R)
2. 콘솔에서 "excluding deleted" 로그 확인
3. is_deleted 필드 수동 확인 (API 테스트)
```

### 문제 3: 체크박스 스타일 깨짐
**증상**: 시간대 체크박스가 세로로 나열됨  
**원인**: CSS 로드 실패  
**해결**:
```
1. Network 탭에서 css/admin.css 로드 확인
2. 캐시 삭제 후 재접속
3. .checkbox-grid 스타일 확인
```

## 📌 알려진 제한사항

1. **시간대 수정 시 기존 신청 영향**
   - 기존에 승인된 신청자는 변경된 시간대 영향 없음
   - 시간대 변경 전에 관리자가 신청자에게 공지 필요

2. **문의 수정 시 간단한 prompt 사용**
   - 현재: `prompt()` 함수 사용 (단순 텍스트 입력)
   - 향후 개선: 모달 기반 rich text 에디터 추가 예정

3. **soft delete의 영구성**
   - is_deleted=true 처리된 데이터는 UI에서 복구 불가
   - 데이터베이스에는 남아있음 (관리자 직접 수정 가능)
