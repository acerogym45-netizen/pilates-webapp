# v12.1.4 레슨 해지 프로그램 목록 동기화

## 🔄 변경 개요 (2024-03-20)

### 📋 문제 상황
**레슨 해지 신청 페이지의 프로그램 목록이 실제 등록된 프로그램과 일치하지 않음**

#### 이전 문제점:
1. **하드코딩된 프로그램 목록**:
   ```html
   <select id="cancelLessonType">
       <option value="화&목 6:1 그룹수업">화&목 6:1 그룹수업</option>
       <option value="수&금 6:1 그룹수업">수&금 6:1 그룹수업</option>
       <option value="2:1 듀엣레슨">2:1 듀엣레슨</option>
       <option value="1:1 개인레슨">1:1 개인레슨</option>
   </select>
   ```

2. **신청서와 불일치**:
   - 신청서: DB에서 동적으로 프로그램 로드 ✅
   - 해지 모달: 고정된 4개 프로그램만 표시 ❌

3. **프로그램명 변경 시 문제**:
   - 관리자가 프로그램명을 수정 (예: "화&목 6:1 그룹수업(12시&18시 3.23 12:00시 오픈 예정)")
   - 해지 모달에는 여전히 구 명칭 표시
   - 신청자와 해지자의 `lesson_type` 불일치 발생

## 🛠️ 수정 사항

### 1. JavaScript 함수 수정 - `showCancellationForm()`

**파일**: `js/main.js` (Line 791-844)

```javascript
// ❌ 수정 전
function showCancellationForm() {
    document.getElementById('cancellationModal').classList.add('active');
}

// ✅ 수정 후
async function showCancellationForm() {
    // Load program list dynamically
    await populateCancellationPrograms();
    
    document.getElementById('cancellationModal').classList.add('active');
}

// 새로운 함수 추가: 프로그램 목록 동적 로드
async function populateCancellationPrograms() {
    try {
        const complexCode = complexContext.getComplexCode();
        if (!complexCode) {
            console.warn('⚠️ Complex code not available for cancellation programs');
            return;
        }
        
        console.log('📋 Loading programs for cancellation modal...');
        
        // Fetch active programs
        const response = await fetch('tables/programs?limit=100&sort=display_order');
        const result = await response.json();
        const programs = result.data || [];
        
        // Filter by complex and active status
        const complexPrograms = programs.filter(p => 
            p.complex_id === complexCode && p.is_active
        );
        
        console.log(`✅ Found ${complexPrograms.length} active programs for cancellation`);
        
        const selectElement = document.getElementById('cancelLessonType');
        
        // Clear existing options except the first one (placeholder)
        selectElement.innerHTML = '<option value="">선택하세요</option>';
        
        // Add program options
        complexPrograms.forEach(program => {
            const option = document.createElement('option');
            option.value = program.program_name;
            option.textContent = program.program_name;
            selectElement.appendChild(option);
        });
        
        if (complexPrograms.length === 0) {
            const option = document.createElement('option');
            option.value = '';
            option.textContent = '등록된 프로그램이 없습니다';
            option.disabled = true;
            selectElement.appendChild(option);
        }
        
    } catch (error) {
        console.error('❌ Failed to load programs for cancellation:', error);
        alert('프로그램 목록을 불러오는데 실패했습니다.');
    }
}
```

**효과**:
- ✅ 해지 모달 열 때마다 최신 프로그램 목록을 DB에서 로드
- ✅ 신청서와 동일한 프로그램명 사용
- ✅ 프로그램 추가/삭제/수정 시 자동 반영

## 🔍 문제 해결 가이드

### Q1. 프로그램 목록이 비어있어요 ("선택하세요"만 보임)
```
A: DB에 활성화된 프로그램이 없을 수 있습니다.
1. 관리자 페이지 → 프로그램 관리
2. 활성화된 프로그램 확인
3. 프로그램이 없으면 새로 추가
4. 입주민 페이지에서 F5 새로고침
```

### Q2. "등록된 프로그램이 없습니다" 메시지가 떠요
```
A: 현재 단지에 활성화된 프로그램이 없습니다.
1. 관리자 페이지 → 프로그램 관리
2. 프로그램 추가 또는 기존 프로그램 활성화
3. complex_id가 현재 단지와 일치하는지 확인
```

### Q3. 콘솔에 "Complex code not available" 경고가 떠요
```
A: 단지 정보가 로드되기 전에 해지 모달을 열었습니다.
1. 페이지 완전히 로드될 때까지 대기
2. 강력 새로고침 (Ctrl+Shift+R)
3. 여전히 안 되면 URL에 ?complex=cheongju-sk 파라미터 확인
```

### Q4. 신청서에는 프로그램이 보이는데 해지 모달에는 안 보여요
```
A: JavaScript 오류가 발생했을 수 있습니다.
1. F12 → Console 탭에서 에러 확인
2. 빨간색 에러 메시지 있으면 스크린샷 전달
3. 배포가 제대로 안 됐을 수도 있음 → 재배포 시도
```
