# 🚨 긴급 배포 및 캐시 해결 가이드

## ❗ 현재 상황
사용자가 **"다시 표시"를 클릭했지만 여전히 입주민 페이지에 문의가 안 보인다**고 보고함.

## 🔍 문제 원인 분석

### ✅ 코드는 이미 수정됨
```javascript
// js/main.js (Line 686-690)
const publicInquiries = inquiries.filter(inq => 
    inq.is_public && 
    inq.complex_id === complexId &&
    !inq.is_hidden &&      // ✅ 숨김 필터링
    !inq.is_deleted        // ✅ 삭제 필터링
);
```

```javascript
// js/admin-inquiry.js (Line 484-493)
const updateData = {
    ...inquiry,
    is_hidden: newStatus
};

delete updateData.gs_project_id;    // ✅ 시스템 필드 제거
delete updateData.gs_table_name;
delete updateData.updated_at;
```

### ❌ 가능한 문제점
1. **배포가 아직 안 됨** → 이전 버전 코드가 실행 중
2. **브라우저 캐시** → 오래된 JavaScript 파일 사용 중
3. **DB 업데이트 실패** → `is_hidden` 값이 실제로 변경되지 않음

## 🚀 해결 방법 (단계별 체크리스트)

### 1단계: 배포 확인 및 재배포 ✅

#### A. Publish 탭에서 배포
```
1. 프로젝트 상단 "Publish" 탭 클릭
2. "Re-deploy" 또는 "Deploy" 버튼 클릭
3. ⚠️ 절대 "Delete data" 옵션 체크 금지
4. 배포 완료 메시지 대기 (30-60초)
5. 배포 URL 복사
```

#### B. 배포 성공 확인
```
✅ "Deployment successful" 메시지 확인
✅ 배포 URL 접속 가능 확인
✅ 배포 시간 확인 (최근 시간이어야 함)
```

---

### 2단계: 브라우저 캐시 완전 삭제 🧹

#### A. 강력 새로고침 (가장 먼저 시도)
```
Windows: Ctrl + Shift + R
Mac: Cmd + Shift + R
```

#### B. 브라우저 캐시 완전 삭제
**Chrome/Edge 기준**:
```
1. F12 키 (개발자 도구 열기)
2. 상단 메뉴에서 "Network" 탭 클릭
3. "Disable cache" 체크박스 체크
4. 페이지 새로고침 (F5)
```

**또는**:
```
1. 브라우저 설정 → 개인정보 및 보안
2. "인터넷 사용 기록 삭제"
3. 시간 범위: "전체 기간"
4. "캐시된 이미지 및 파일" 체크
5. "데이터 삭제" 클릭
```

#### C. 시크릿/프라이빗 모드 테스트
```
Chrome: Ctrl + Shift + N
Firefox: Ctrl + Shift + P
Edge: Ctrl + Shift + N

→ 시크릿 모드에서 배포 URL 접속
→ 캐시 없이 최신 코드 실행됨
```

### 3단계: DB 상태 직접 확인 🔍

#### A. 관리자 페이지에서 확인

2. F12 (개발자 도구) → Console 탭

#### C. 문제 발견 시 수동 수정
```javascript
// 특정 문의의 is_hidden 값을 false로 강제 수정
const inquiryId = 'INQUIRY_ID_HERE';  // 문의 ID 입력

fetch(`tables/pilates_inquiries/${inquiryId}`)
  .then(r => r.json())
  .then(inquiry => {
    const updateData = { ...inquiry, is_hidden: false };
    delete updateData.gs_project_id;
    delete updateData.gs_table_name;
    delete updateData.updated_at;
    
    return fetch(`tables/pilates_inquiries/${inquiryId}`, {
      method: 'PUT',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(updateData)
    });
  })
  .then(r => r.json())
  .then(result => console.log('✅ Updated:', result));
```

## 🎯 문제별 해결 방법

### 문제 1: "다시 표시" 클릭 시 콘솔에 에러
```javascript
// Console에서 에러 확인
❌ PUT failed
❌ Failed to 숨김 해제 inquiry

→ 해결: 
1. 배포가 안 됐을 가능성 → 2단계 재배포
2. 네트워크 문제 → 인터넷 연결 확인
```

### 문제 2: "다시 표시" 성공 메시지는 뜨는데 입주민 페이지에 안 보임
```javascript
// Console에서 확인
✅ Inquiry 숨김 해제 successfully

→ 해결:
1. 입주민 페이지가 오래된 캐시 사용 중
2. 입주민 페이지에서 Ctrl+Shift+R (강력 새로고침)
3. 시크릿 모드에서 입주민 페이지 접속
```

### 문제 3: DB에 is_hidden=false인데도 안 보임
```javascript
// Console 확인
is_public: true ✅
is_hidden: false ✅
is_deleted: false ✅
complex_id: 'cheongju-sk' ✅

→ 해결:
1. js/main.js의 loadPublicInquiries() 함수가 오래된 버전
2. 배포 재확인
3. 캐시 완전 삭제 후 재시도
```

### 문제 4: 관리자에서는 보이는데 입주민에서만 안 보임
```javascript
// 두 페이지가 다른 필터링 로직 사용 중

→ 해결:
1. 입주민 페이지 URL에 ?complex= 파라미터 확인
2. complex_id 불일치 가능성 → DB에서 문의의 complex_id 확인
3. 콘솔에서 필터링 로직 직접 실행 (4단계 B 참고)
```

## 🆘 그래도 안 되면...

### 마지막 수단: 하드 리로드 + 캐시 완전 삭제
```
1. 모든 브라우저 탭 닫기
2. 브라우저 완전 종료
3. 브라우저 캐시 폴더 수동 삭제
   - Windows: C:\Users\USERNAME\AppData\Local\Google\Chrome\User Data\Default\Cache
   - Mac: ~/Library/Caches/Google/Chrome/Default/Cache
4. 브라우저 재시작
5. 시크릿 모드에서 접속
```

### 긴급 핫픽스: 버전 번호 강제 변경
```javascript
// js/main.js 맨 위에 추가 (캐시 무력화)
const VERSION = 'v12.1.3-' + Date.now();
console.log('🚀 App version:', VERSION);
```

이렇게 하면 브라우저가 파일을 새 버전으로 인식하여 캐시를 사용하지 않습니다.
