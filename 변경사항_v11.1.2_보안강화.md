# 📋 변경사항 v11.1.2 – 단지 관리 페이지 비밀번호 보안 강화

**날짜**: 2026-03-19

**작업 유형**: 보안 강화, 버그 수정

## 🐛 문제 상황

### 발견된 버그
**단지 관리 페이지 자동 로그인 문제**

**증상**:
- `admin-complex.html` 접속 시 비밀번호 화면이 1초만 표시되고 사라짐
- 자동으로 단지 관리 메인 화면으로 진입
- 비밀번호 없이 접근 가능한 보안 취약점

**사용자 경험**:
```
admin-complex.html 접속
    ↓
[비밀번호 화면 표시]
    ↓
[1초 후 자동으로 사라짐] ❌
    ↓
[메인 화면 자동 진입] ❌
```

---

## 🔍 원인 분석

### 1. sessionStorage 자동 인증 메커니즘
**v11.1.1 코드**:
```javascript
document.addEventListener('DOMContentLoaded', function() {
    if (sessionStorage.getItem('complexAuthToken') === 'authenticated') {
        showMainContent();  // ← 자동 로그인
    } else {
        // 비밀번호 화면 표시
    }
});
```

### 2. 문제의 흐름
1. **첫 방문**: 비밀번호 입력 → `sessionStorage.setItem('complexAuthToken', 'authenticated')`
2. **재방문**: sessionStorage 체크 → 값이 'authenticated' → 자동 로그인
3. **보안 문제**: 브라우저를 닫지 않으면 계속 자동 로그인

### 3. sessionStorage 특성
- **유효 기간**: 브라우저 탭/창을 닫을 때까지 유지
- **공유 범위**: 같은 탭 내에서만 유효 (다른 탭과 공유 안 됨)
- **위험성**: 브라우저를 장시간 열어두면 누구나 접근 가능

## ✅ 해결 방법

### sessionStorage 자동 인증 제거

**변경 전 (v11.1.1)**:
```javascript
document.addEventListener('DOMContentLoaded', function() {
    // sessionStorage 체크 → 자동 로그인 허용
    if (sessionStorage.getItem('complexAuthToken') === 'authenticated') {
        showMainContent();
    } else {
        document.getElementById('passwordScreen').style.display = 'flex';
        document.getElementById('mainContent').style.display = 'none';
    }
});

function checkPassword() {
    if (password === MASTER_PASSWORD) {
        sessionStorage.setItem('complexAuthToken', 'authenticated'); // ← 저장
        showMainContent();
    }
}
```

**변경 후 (v11.1.2)**:
```javascript
document.addEventListener('DOMContentLoaded', function() {
    // Always require password entry - 매번 비밀번호 입력 필수
    document.getElementById('passwordScreen').style.display = 'flex';
    document.getElementById('mainContent').style.display = 'none';
    
    const input = document.getElementById('passwordInput');
    if (input) {
        input.focus();
    }
});

function checkPassword() {
    if (password === MASTER_PASSWORD) {
        // sessionStorage 저장 제거 - 매번 인증 필요
        showMainContent();
    }
}
```

## 🔐 보안 이점

### 1. 세션 하이재킹 방지
- **Before**: sessionStorage 토큰 탈취 시 무제한 접근 가능
- **After**: 매번 비밀번호 입력 → 토큰 없음 → 하이재킹 불가능

### 2. 물리적 접근 제어
- **Before**: 컴퓨터를 잠시 자리 비운 사이 타인 접근 가능
- **After**: 페이지를 나갔다가 돌아오면 다시 인증 필요

### 3. 민감한 관리 기능 보호
- **단지 관리 페이지**: 단지 추가/삭제, 비밀번호 설정 등 민감한 기능
- **필요성**: 최고 수준의 보안 → 매번 인증이 적절

### 4. 의도적 접근만 허용
- 실수로 페이지 열었을 때 자동 진입 방지
- 관리자가 명확히 의도하고 비밀번호 입력 시에만 접근
