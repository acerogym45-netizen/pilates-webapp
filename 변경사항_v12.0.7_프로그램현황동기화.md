# v12.0.7 - 📊 프로그램 현황 실시간 동기화

**날짜**: 2026-03-20

## 🎯 해결된 문제

### 문제 상황
- **관리자 페이지**: 승인된 신청자가 제대로 표시됨
- **입주민 페이지**: 프로그램 드롭다운에서 현재 모집 인원이 표시되지 않음
- **근본 원인**: `programs.current_count` 필드가 실제 승인된 신청자 수와 동기화되지 않음

### 사용자 보고
> "신청 관리 현황에는 적용이 된 것 같은데 입주민 페이지에서 현황 매칭이 안되고있어"

    // 1. 프로그램 로드
    const response = await fetch('tables/programs?limit=100');
    
    // 2. 승인된 신청 데이터 로드
    const contractsResponse = await fetch('tables/pilates_contracts?limit=1000');

- ✅ `pilates_contracts` 테이블에서 승인된(`status: 'approved'`) 신청 데이터 조회
- ✅ 각 프로그램(`program_name`)별 승인자 수를 실시간으로 계산
- ✅ 단지별(`complex_id`) 필터링 적용
- ✅ 콘솔 로그로 각 프로그램의 현황 출력

function handlePage1Submit(e) {
    e.preventDefault();
    
    const lessonTypeSelect = document.getElementById('lessonType');
    const selectedOption = lessonTypeSelect.options[lessonTypeSelect.selectedIndex];
    
    // 정원 체크
    if (selectedOption && selectedOption.dataset.maxCapacity) {
        const currentCount = parseInt(selectedOption.dataset.currentCount) || 0;
        const maxCapacity = parseInt(selectedOption.dataset.maxCapacity) || 0;

            return; // 신청 진행 중단
        }

- **변경 사항**: JavaScript 로직만 수정 (클라이언트 사이드)
- **데이터베이스**: 전혀 변경되지 않음
- **기존 데이터**: 모두 보존됨

  - `pilates_contracts` 테이블: 88개 신청 데이터 유지
  - `pilates_cancellations`: 기존 데이터 유지
  - `pilates_inquiries`: 기존 데이터 유지
  - 기타 모든 테이블 유지

### 동작 방식
- `current_count`는 **읽기 전용** (페이지 로드 시 실시간 계산)
- DB의 `programs.current_count` 값은 무시됨
- 항상 `pilates_contracts` 테이블의 승인된 신청 수를 카운트

## ✅ 결론

- **문제**: 입주민 페이지에서 프로그램 현황이 관리자 페이지와 동기화되지 않음

- **해결**: 페이지 로드 시 `pilates_contracts` 테이블에서 승인된 신청 수를 실시간 계산
