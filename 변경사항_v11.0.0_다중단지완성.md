# 🔄 변경사항 v11.0.0 - 다중 단지 시스템 완성 (URL 파라미터 방식)

## 📅 업데이트 일시: 2026-03-19

---

## ✨ 핵심 기능: 다중 단지 지원 시스템

### 개요
**하나의 웹사이트에서 여러 단지를 독립적으로 관리**할 수 있는 시스템을 구축했습니다!

단지를 추가하면:
1. 해당 단지만의 고유 URL 자동 생성
2. QR 코드 자동 생성
3. 단지별 데이터 완전 분리
4. 단지별 브랜딩 자동 적용

---

## 🏗️ 시스템 아키텍처

### URL 기반 멀티 테넌시
```
청주SK뷰자이:  https://your-site.com/?complex=cheongju-sk
서울타워:     https://your-site.com/?complex=seoul-tower
부산마린시티: https://your-site.com/?complex=busan-marine
```

각 URL은 **완전히 독립된 웹사이트**처럼 동작합니다!

### 1️⃣ 테이블 스키마 업데이트

모든 주요 테이블에 `complex_id` 필드 추가:

|--------|-----------------|----------------|-----------|
| pilates_contracts | 14 | 15 | +complex_id |
| pilates_cancellations | 11 | 12 | +complex_id |
| pilates_inquiries | 12 | 13 | +complex_id |
| instructors | 10 | 11 | +complex_id |
| notices | 7 | 8 | +complex_id |
| complex_settings | 12 | 12 | (기존) |

### 2️⃣ 데이터 분리
- 각 레코드는 하나의 단지에 속함
- 단지 A의 신청자는 단지 B에서 보이지 않음
- 공지사항, 강사도 단지별로 독립

## 🆕 새로운 파일 및 기능

### 1. `js/complex-context.js` (신규)
**단지 컨텍스트 관리자**

```javascript
// 주요 기능:
- URL 파라미터에서 단지 코드 읽기
- 단지 설정 로드
- 브랜딩 자동 적용 (로고, 색상, 제목)
- 전역적으로 현재 단지 정보 제공
```

**클래스 구조**:
```javascript
class ComplexContext {
    getComplexCodeFromURL()      // URL에서 complex 파라미터 읽기
    loadComplexSettings()         // 단지 설정 로드
    getComplexId()                // 현재 단지 ID 반환
    applyBranding()               // 브랜딩 적용
    getFilterParams()             // API 필터 파라미터
}
```

## 🎨 입주민 페이지 변경사항

### 수정된 파일
1. **index.html**
   - `js/complex-context.js` 스크립트 추가
   
2. **js/main.js**
   - 초기화 시 단지 컨텍스트 먼저 로드
   - 모든 데이터 제출 시 `complex_id` 포함
   - 공지사항/문의 로드 시 단지별 필터링

3. **instructors.html**
   - 강사 로드 시 단지별 필터링

### 동작 방식
```javascript
// 1. 페이지 로드
https://site.com/?complex=cheongju-sk 접속

// 2. JavaScript가 URL 파라미터 읽기
const complexCode = 'cheongju-sk';

// 3. 단지 설정 로드
complex_settings 테이블에서 cheongju-sk 찾기

// 4. 브랜딩 적용
- 페이지 제목: "청주SK뷰자이"
- 로고 표시 (있으면)
- 주요 색상 변경

// 5. 데이터 필터링
- 공지사항: complex_id = '단지ID'인 것만 표시
- 강사: complex_id = '단지ID'인 것만 표시

// 6. 신청서 제출
{
    ...formData,
    complex_id: '단지ID'  // 자동 추가
}
```

## 📋 사용 시나리오

### 시나리오 1: 새 단지 추가

**관리자 작업**:
1. admin-complex.html 접속 (비밀번호: master2026)
2. [단지 추가] 버튼 클릭
3. 정보 입력:
   ```
   단지명: 서울타워
   단지 코드: seoul-tower (영문/하이픈)
   연락처: 02-XXXX-XXXX
   주소: 서울시 강남구
   로고: (선택)
   색상: #3498db
   프로그램: 화&목 6:1 그룹수업, 수&금 6:1 그룹수업
   활성화: ✓
   ```
4. [저장] 클릭
5. [QR 코드 생성] 버튼 클릭
6. QR 코드 인쇄 또는 다운로드

**입주민 사용**:
1. QR 코드 스캔
2. `https://your-site.com/?complex=seoul-tower` 자동 이동
3. "서울타워" 전용 페이지 표시
4. 레슨 신청, 문의 작성

---

### 시나리오 2: 단지별 관리

**청주SK뷰자이 관리자**:
- URL: `https://your-site.com/?complex=cheongju-sk`
- 청주SK뷰자이 데이터만 표시
- 다른 단지 데이터는 보이지 않음

**서울타워 관리자**:
- URL: `https://your-site.com/?complex=seoul-tower`
- 서울타워 데이터만 표시
- 청주 데이터는 보이지 않음

## 🎯 데이터 흐름

### 신청서 제출
```
입주민 (서울타워)
    ↓
/?complex=seoul-tower 접속
    ↓
JavaScript: complexContext.getComplexId() → "서울타워ID"
    ↓
신청서 작성 & 제출
    ↓
POST /tables/pilates_contracts
{
    complex_id: "서울타워ID",
    name: "홍길동",
    ...
}
    ↓
데이터베이스 저장 (complex_id 포함)
```

### 데이터 조회
```
관리자 (청주SK뷰자이)
    ↓
admin-main.html 접속
    ↓
GET /tables/pilates_contracts
    ↓
JavaScript 필터링: 
filter(c => c.complex_id === "청주ID")
    ↓
청주SK뷰자이 신청만 표시
```

### 단지 코드 네이밍 규칙
```
✅ 좋은 예:
- cheongju-sk
- seoul-tower
- busan-marine-city

- 청주SK (한글 불가)
- Seoul Tower (공백 불가)
```

1. **안내 포스터**: 단지 입구에 QR 코드 게시
2. **가정 통신문**: 우편함에 QR 코드 전단지
3. **게시판**: 엘리베이터에 QR 코드 부착

### URL 공유 방법
1. **카카오톡**: URL 복사 → 단지 채팅방에 공유
2. **문자**: URL 복사 → 입주민 전체 문자 발송
3. **이메일**: URL 포함한 안내 메일 발송
