## 🐛 발견된 문제

### **증상**
- ✅ **입주자 페이지**: 프로그램 드롭다운에 4개 프로그램 정상 표시
- ❌ **관리자 프로그램 관리 페이지**: "등록된 프로그램이 없습니다" 표시

### **가능한 원인**
1. `admin-programs.html` 페이지가 올바른 `complex_code`를 사용하지 못함
2. DB에서 데이터를 가져왔지만 필터링 실패
3. API 응답 구조 문제
4. 로그 부족으로 원인 파악 어려움

## ✅ 해결 방법

### **1. 상세 디버깅 로그 추가**

**파일**: `js/admin-programs.js`

#### **`loadPrograms()` 함수 개선**

**변경 전**:
```javascript
const response = await fetch(`${TABLES_ENDPOINT}/${TABLE_NAME}?limit=100&sort=display_order`);
const result = await response.json();

// Filter by current complex
allPrograms = (result.data || []).filter(p => p.complex_id === currentComplexId);

console.log(`📋 Loaded ${allPrograms.length} programs for complex ${currentComplexId}`);
```

**변경 후**:
```javascript
console.log(`🔍 Fetching programs for complex: ${currentComplexId}`);

const response = await fetch(`${TABLES_ENDPOINT}/${TABLE_NAME}?limit=100&sort=display_order`);
const result = await response.json();

console.log(`📦 Total programs fetched: ${result.data ? result.data.length : 0}`, result.data);

// Filter by current complex with detailed logging
allPrograms = (result.data || []).filter(p => {
    const match = p.complex_id === currentComplexId;
    console.log(`  - ${p.program_name}: complex_id="${p.complex_id}" ${match ? '✅' : '❌'}`);
    return match;
});

console.log(`📋 Loaded ${allPrograms.length} programs for complex ${currentComplexId}`, allPrograms);
```

**추가된 로그**:
1. 🔍 어떤 complex_code로 검색하는지
2. 📦 API에서 가져온 전체 프로그램 수
3. 각 프로그램의 complex_id와 매칭 여부 (✅/❌)
4. 최종 필터링된 프로그램 배열

#### **`getDefaultComplexId()` 함수 개선**

**변경 전**:
```javascript
async function getDefaultComplexId() {
    try {
        const response = await fetch(`${TABLES_ENDPOINT}/complex_settings?limit=1`);
        const result = await response.json();
        if (result.data && result.data.length > 0) {
            return result.data[0].complex_code;
        }
    } catch (error) {
        console.error('Error getting default complex:', error);
    }
    return 'cheongju-sk';
}
```

**변경 후**:
```javascript
async function getDefaultComplexId() {
    try {
        console.log('🔍 Fetching default complex...');
        const response = await fetch(`${TABLES_ENDPOINT}/complex_settings?limit=1`);
        const result = await response.json();
        console.log('📦 Complex settings result:', result);
        
        if (result.data && result.data.length > 0) {
            const complexCode = result.data[0].complex_code;
            console.log(`✅ Using complex_code: ${complexCode}`);
            return complexCode;
        }
    } catch (error) {
        console.error('❌ Error getting default complex:', error);
    }
    console.log('⚠️ Using fallback: cheongju-sk');
    return 'cheongju-sk';
}
```

**추가된 로그**:
1. 🔍 complex_settings 데이터 요청 시작
2. 📦 API 응답 전체 출력
3. ✅ 사용할 complex_code 명시
4. ⚠️ 폴백 값 사용 시 경고

## 🔍 디버깅 가이드

### **1. 브라우저 콘솔 확인 방법**
```
1. Chrome/Edge: F12 → Console 탭
2. Firefox: F12 → 콘솔 탭
3. Safari: Cmd+Opt+C → 콘솔
```

### **2. 확인할 항목**
| 로그 메시지 | 확인 사항 | 예상 값 |
|------------|----------|---------|
| `Using complex_code:` | complex_code가 무엇인지 | `cheongju-sk` |
| `Total programs fetched:` | DB에 프로그램이 있는지 | `8` |
| `complex_id="..."` | 각 프로그램의 complex_id | `cheongju-sk` |
| `Loaded X programs` | 필터링 후 개수 | `8` |

### **3. 문제별 해결 방법**

#### **문제 A: `Using complex_code: undefined`**
**원인**: complex_settings 테이블이 비어있음  
**해결**: Publish 탭에서 데이터 확인, complex_settings에 데이터 추가

#### **문제 B: `Total programs fetched: 0`**
**원인**: programs 테이블이 비어있음  
**해결**: 프로그램 데이터가 삭제됨, 다시 추가 필요

#### **문제 C: `complex_id="cheongju-sk" ❌` (모두 ❌)**
**원인**: currentComplexId와 programs.complex_id 불일치  
**해결**: 
- `complex_settings`의 `complex_code` 확인
- `programs`의 `complex_id` 확인
- 두 값이 일치하도록 수정
